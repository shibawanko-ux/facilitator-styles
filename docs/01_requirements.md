# ファシリスタイル 要件定義書

## 1. プロジェクト概要

### 1.1 アプリ名

**ファシリスタイル**

### 1.2 目的

ファシリテーターとしての自分のスタイル・傾向を4つの軸で可視化し、自己理解と成長のヒントを提供する

### 1.3 ターゲットユーザー

- ファシリテーションに興味がある人
- ワークショップや会議のファシリテーターを担当する人
- 自分のファシリテーションスタイルを客観的に知りたい人
- コーファシリテーションを行うチーム

### 1.4 公開範囲

一般公開

### 1.5 ブランド

- **ロゴ**: awareness=design のロゴ画像を使用
- **提供元**: awareness=design

## 2. 診断システム設計

### 2.1 4つの軸（二分法）

| 軸 | 両極 | サブタイトル |
|---|---|---|
| 介入スタイル | 触発型 ⇔ 見守型 | （場に働きかけ火をつける）⇔（沈黙を活かし待つ） |
| 知覚対象 | 観察型 ⇔ 洞察型 | （言葉・事実を拾う）⇔（空気・流れを読む） |
| 判断基準 | 目的型 ⇔ 関係型 | （ゴール達成を優先）⇔（納得感・関係性を優先） |
| 場への関わり | 設計型 ⇔ 即興型 | （計画通りに進める）⇔（その場で柔軟に変える） |

### 2.2 結果表示方式

- 4軸の組み合わせで**16タイプに分類**し、タイプ名（ニックネーム）を表示
- 各軸のスコアをグラフで可視化
- 軸ごとの強み・弱み・成長ヒントを表示

### 2.3 16タイプ一覧

#### 触発型ベース（8タイプ）

| 組み合わせ | ニックネーム | イメージ |
|---|---|---|
| 触発×観察×目的×設計 | **場の指揮者** | 計画的に場をリード、ゴールに導く |
| 触発×観察×目的×即興 | **推進のエンジン** | 事実を見ながら柔軟にゴールへ加速 |
| 触発×観察×関係×設計 | **場の調律師** | 人と事実を丁寧につなげる |
| 触発×観察×関係×即興 | **共感のスパーク** | その場の声を拾って火をつける |
| 触発×洞察×目的×設計 | **戦略のナビゲーター** | 先を読んで道筋を示す |
| 触発×洞察×目的×即興 | **直感の開拓者** | 空気を読んで新しい道を切り拓く |
| 触発×洞察×関係×設計 | **場の演出家** | 感情と流れを設計して盛り上げる |
| 触発×洞察×関係×即興 | **ムードメーカー** | 空気を読んで場に火を灯す |

#### 見守型ベース（8タイプ）

| 組み合わせ | ニックネーム | イメージ |
|---|---|---|
| 見守×観察×目的×設計 | **静かな舵取り** | 淡々と事実を積み上げゴールへ |
| 見守×観察×目的×即興 | **柔軟な記録者** | 発言を拾いながら軌道修正 |
| 見守×観察×関係×設計 | **信頼の土台** | 安心感を作り、関係を育てる |
| 見守×観察×関係×即興 | **寄り添う聴き手** | 声を拾い、場に安心を与える |
| 見守×洞察×目的×設計 | **静かな戦略家** | 先を見据えて裏から支える |
| 見守×洞察×目的×即興 | **流れを読む羅針盤** | 空気を読みつつ柔軟に導く |
| 見守×洞察×関係×設計 | **場の守り人** | 深い洞察で関係性を守る |
| 見守×洞察×関係×即興 | **静かな共鳴者** | 空気を感じ、寄り添う存在 |

### 2.4 質問設計

- **問数**: 32問（各軸8問）
- **回答形式**: 5段階リッカートスケール
  - 1: とてもAに近い
  - 2: ややAに近い
  - 3: どちらとも言えない
  - 4: ややBに近い
  - 5: とてもBに近い

### 2.5 スコアリングロジック

- 各軸のスコアを合計（8問 × 1〜5点 = 8〜40点）
- スコアに応じて、どちらの傾向が強いかを判定
- 中間付近は「バランス型」として表示
- 4軸の組み合わせで16タイプのうち1つを決定

## 3. 機能要件

### 3.1 画面構成

```
[トップページ] → [質問画面(32問)] → [結果画面] → [シェア/保存]
```

### 3.2 各画面の詳細

#### トップページ

**ヒーローセクション**
- awareness=designロゴ画像
- アプリ名「ファシリスタイル」
- サブタイトル「ファシリテーター診断」
- アプリの説明（何が分かるか）
- 4つの軸の概要（カード形式で表示）
- 所要時間の目安（約5分）
- 「診断を始める」ボタン
- 注意書き（ボタンの下に配置）

**タイプ一覧セクション**
- 「ファシリテータータイプ」セクション
- 触発型ファシリテーター
  - 推進者グループ（4タイプ）
  - 共感者グループ（4タイプ）
- 見守型ファシリテーター
  - 戦略家グループ（4タイプ）
  - 守護者グループ（4タイプ）
- 各タイプをカード形式で表示（タイプ名、キャッチコピー）

#### 質問画面

- プログレスバー（進捗表示）
- 質問番号（例：質問 1 / 32）
- 質問文
- 選択肢の両極（AとBのラベル）
- 5段階の回答ボタン（1〜5）
- 「戻る」「次へ」ボタン（角丸デザイン）
- フェードインアニメーション

#### 結果画面

**セクション1: あなたのタイプ**

- **タイプ名（ニックネーム）** を大きく表示（例：「場の指揮者」）
- キャッチコピー
- タイプの詳細説明（2〜3段落）
- 得意な場面（タグ形式で表示）
- フェードインアニメーション

**セクション2: 性格特性**

- 4軸のスコアをバーで可視化
- 各軸の傾向ラベル（「触発型寄り」「ほぼ中間」等）
- 傾向のパーセンテージ表示
- 各軸の簡易説明文

**セクション3: 強み・注意点**

- 2カラムレイアウト（強み / 注意点）
- 各軸から代表的な項目を抽出してカード形式で表示
- 各項目にタイトルと説明文

**セクション4: 各軸の詳細分析**

- アコーディオン形式（クリックで展開）
- 軸ごとに以下を表示：
  - 傾向の詳細説明（100〜150字）
  - 強み（カード形式、タイトル＋説明文）
  - 注意点・弱み（カード形式、タイトル＋説明文）
  - 成長のヒント（リスト形式）

**セクション5: コーファシリとの組み合わせヒント**

- タブ切り替え（メイン / サブ）
- メインファシリを担当するときのヒント
- サブファシリを担当するときのヒント
- 補い合える傾向の説明

**セクション6: シェア・保存**

- SNSシェアボタン（X/Twitter, Facebook, LINE）
- 結果画像ダウンロードボタン

## 4. 非機能要件

### 4.1 プラットフォーム

- Webアプリ（SPA）
- スマートフォン対応（レスポンシブデザイン）

### 4.2 デザイン方針

- **スタイル**: ミニマル・フラットデザイン
- **余白**: 多めに取ったクリーンなUI
- **タイポグラフィ**: Noto Sans JP、読みやすいサイズ、文字間隔を広めに
- **アニメーション**: 
  - フェードイン（fadeIn, fadeInUp）
  - スケールアニメーション（ホバー時）
  - トランジション（300ms ease-out）
- **カラーパレット**:
  - プライマリ: スレート系（slate-600〜700）
  - アクセント: エメラルド系（emerald-500〜600）
  - サブ: スカイ系（sky-500〜600）、ティール系（teal-500〜600）
  - 背景: 白、グレー50
- **カード**: 角丸2xl、薄いボーダー、ホバーで軽いシャドウ
- **ボタン**: 角丸フル（rounded-full）、ホバーでスケールアップ

### 4.3 技術スタック

- **フロントエンド**: React 18 + Vite 5 + TypeScript
- **スタイリング**: Tailwind CSS 3
- **画像生成**: html2canvas（結果画像保存用）
- **ホスティング**: Vercel
- **アクセス解析**: Google Analytics（want要件）

## 9. 使い方

### 開発環境での起動

```bash
# プロジェクトディレクトリに移動
cd app_Facilitator_type_diagnosis

# 依存関係をインストール
npm install

# 開発サーバーを起動
npm run dev
```

ブラウザで `http://localhost:5173` にアクセスして確認できます。

詳細は `docs/07_startup_guide.md` を参照してください。

### 本番ビルド

```bash
npm run build
```

`dist/` フォルダに本番用ファイルが生成されます。

### デプロイ

詳細は `docs/06_deploy_guide.md` を参照してください。
