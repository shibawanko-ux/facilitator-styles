# ファシリスタイル 要件定義書

## 1. プロジェクト概要

### 1.1 アプリ名

**ファシリスタイル**

### 1.2 目的

ファシリテーターとしての自分のスタイル・傾向を4つの軸で可視化し、自己理解と成長のヒントを提供する

### 1.3 ターゲットユーザー

- ファシリテーションに興味がある人
- ワークショップや会議のファシリテーターを担当する人
- 自分のファシリテーションスタイルを客観的に知りたい人
- コーファシリテーションを行うチーム

### 1.4 公開範囲

一般公開

### 1.5 ブランド

- **ロゴ**: awareness=design のロゴ画像を使用
- **提供元**: awareness=design
- **出典**: フッターに「ファシリテーション研究を参考にした診断です」を表示

## 2. 診断システム設計

### 2.1 4つの軸（二分法）

| 軸 | 両極 | サブタイトル |
|---|---|---|
| 介入スタイル | 触発型 ⇔ 見守型 | （場に働きかけ火をつける）⇔（沈黙を活かし待つ） |
| 知覚対象 | 観察型 ⇔ 洞察型 | （言葉・事実を拾う）⇔（空気・流れを読む） |
| 判断基準 | 目的型 ⇔ 関係型 | （ゴール達成を優先）⇔（納得感・関係性を優先） |
| 場への関わり | 設計型 ⇔ 即興型 | （計画通りに進める）⇔（その場で柔軟に変える） |

### 2.1.1 4軸と4つのスタイルの関係（採用済み）

「4つのスタイル」（提案・触発・整理・共感）は、現行4軸の上位コンセプトとして位置づけ。「論理⇔感情」「働きかける⇔受け止める」の2軸で4タイプに大別され、本診断ではこれを4軸でより細かく16スタイルに分けて診断する。詳細は [10_enhancement_requirements.md](10_enhancement_requirements.md) を参照。

**4軸カードの補足文言（実装済み）**

| 軸 | 補足文言 |
|---|----------|
| 介入スタイル | 場に自分から働きかけるか、相手を受け止めるか |
| 知覚対象 | 言葉・事実を拾うか、空気・感情を読むか |
| 判断基準 | ゴール達成を優先するか、納得感・関係性を優先するか |
| 場への関わり | 計画通りに進めるか、その場で柔軟に変えるか |

**16タイプと4つのスタイルの対応**

各タイプの詳細説明（`detailedDescription`）に、該当するスタイル（提案・触発・整理・共感）の特徴を1文程度織り込む。例：場の指揮者→「提案スタイル」、ムードメーカー→「触発スタイル」、静かな舵取り→「整理スタイル」、寄り添う聴き手→「共感スタイル」。

### 2.2 結果表示方式

- 4軸の組み合わせで**16スタイルに分類**し、スタイル名（ニックネーム）を表示
- **表記方針**（2026-01-25）：画面・文言では「16スタイル」「あなたのファシリテータースタイル」「他のスタイルとの相性」「〇〇スタイルでした」など**スタイル**で統一。データ・ドキュメントの見出し（例：2.3 16タイプ一覧）では従来の「タイプ」表記も併用可。
- 各軸のスコアをグラフで可視化
- 軸ごとの強み・弱み・成長ヒントを表示

### 2.4 質問設計

- **問数**: 32問（各軸8問）
- **回答形式**: 6段階スケール（中立なし・どちらかに寄せる）
  - 1: とてもAに近い
  - 2: ややAに近い
  - 3: どちらかといえばA
  - 4: どちらかといえばB
  - 5: ややBに近い
  - 6: とてもBに近い

### 2.5 スコアリングロジック

- 各軸のスコアを合計（8問 × 1〜6点 = 8〜48点）
- 中央28でA/Bを判定（27以下: A寄り、28以上: B寄り）
- 中間付近は「バランス型」として表示
- 4軸の組み合わせで16タイプのうち1つを決定

### 2.6 ユーザーテスト結果と改善方針（2026-01-25）

**観察された課題**

- **回答形式の偏り**: 5段階リッカートで「1」「5」を選ぶ人が少なく、中心の「2」「3」「4」に回答が集中しがち。
- **結果の振り幅**: 上記のため軸スコアが中央付近に集まり、16タイプの結果が偏り、振り幅が小さい傾向。

**採用方針: 案C（6段階化）**

- 1〜3をA側、4〜6をB側とし、中立を廃止。スコア範囲は 8〜48点、中央28でA/Bを判定。
- 詳細な変更履歴は `docs/08_change_proposal_usertest.md` を参照。

### 2.7 傾向キーワード表示（2.7 採用済み）

**表現したい内容**

- 結果画面の**タイプ詳細説明の下**、**「得意な場面」の上**に、型×グループの階層（タイトル・サブタイトル・リード文）＋4象限の図を表示する。
- **4象限の意味を図に表現**する。左上＝**介入スタイル**、左下＝**判断基準**、右上＝**知覚対象**、右下＝**場の関わり**。
- 各象限には**軸名**と診断結果の**型ラベル**（例：見守型、洞察型、目的型、設計型）と、**その型から取ったキーワード4語**を表示する。キーワードは各軸の分析内容（axisContents の強みなど）から抽出する。合計**16語**（4軸×4語）。
- **角Rは使わず表の表現**（枠線で区切ったセル）で表示する。※タイプ名はヒーローセクションで表示するため、このセクションでは重複表示しない。

**実装済み要件**

- **配置**: タイプの詳細説明の直下、「得意な場面」セクションの直上。
- **見出し・階層**: 1) タイトル「あなたのファシリテーション特性」（小さく・太字）→ 2) サブタイトル「〇〇型ファシリテーター × △△グループ」（型×グループを表示。**塊と同色のフォント色**：推進者=red-900、共感者=red-800、戦略家=#1F86C8、守護者=#38a5d8）→ 3) リード文（介入スタイル＋グループの説明を組み合わせた1文）→ 4) 4象限グリッド。
- **リード文**: 型×グループの組み合わせごとに固定文言（例：見守型×守護者＝「静かに見守り、参加者の主体性を引き出し、安心感を与え、関係性を守り育てる」）。
- **内容**: 4象限それぞれに**軸名：型ラベル**を1行で表示（例：介入スタイル：**見守型**）。**キーワード4語**は「・」付き箇条書き（各型に紐づく axisContents 由来）。
- **レイアウト**: 2×2グリッド。左上＝介入スタイル、右上＝知覚対象、左下＝判断基準、右下＝場の関わり。軸色は2.8.3に準拠。
- **表示スタイル**: 角Rなし、`border` で区切った表形式。`typeKeywords.ts` で軸別4語を管理。

### 2.8 傾向キーワード・ファシリテーター特性の表示改善（2.8 要件）

**背景・課題**

- 傾向キーワード図で**タイプ名**（例：寄り添う聴き手）と**強み（キーワード）**の文字が重なって読めない。
- 4象限内で**軸名・型ラベル**（例：介入スタイル、見守型）と**強み4語**の区別がつきにくい（同一フォントサイズ・字重）。
- 4象限および「ファシリテーター特性」セクションに、軸ごとの**色のルール**がなく、視覚的な対応が分かりにくい。

**要件一覧**

| # | 要件 | 詳細・受入条件 |
|---|------|----------------|
| 2.8.1 | タイプ名と強みの重なり解消 | タイプ名と4象限の強みテキストが重ならないようにする。例：タイプ名を図の**上**または**下**に独立して表示するなど、重なりが発生しないレイアウトとする。 |
| 2.8.2 | 4象限内の視覚的階層 | 各象限で**軸名**（介入スタイル・知覚対象・判断基準・場の関わり）および**型ラベル**（見守型・洞察型など）を、**強み4語**より識別しやすくする。文字サイズ・字重・余白を工夫し、タイトルと内容が区別できること。 |
| 2.8.3 | 4軸の色ルール | 4つの軸に固定の色を割り当て、**トップページの4軸説明**・**傾向キーワードの4象限**・**ファシリテーター特性**で同じルールを用いる。**強みセクションは1色（エメラルド）に統一**（2.9 参照）。<br>**割り当て**: 介入スタイル＝赤系統、判断基準＝黄色系統、知覚対象＝緑系統、場の関わり＝青系統。 |
| 2.8.4 | 傾向キーワード図の色付け | 4象限の各セルに、上記2.8.3の軸色を反映する（背景の薄い色または左ボーダーなど）。図の上に見出し「あなたのファシリテーション特性」、サブタイトル（型×グループ）、リード文を表示（2.9 参照）。 |
| 2.8.5 | ファシリテーター特性・トップの色付け | 「ファシリテーター特性」セクション内の4軸（スコアバー・軸名・型ラベル等）と**トップページの4軸説明カード**に、2.8.3と同じ軸色を適用する。結果画面の「強み」は1色に統一（2.9）。 |

**色の対応表（実装用）**

| 軸 | 軸名 | 色系統 | Tailwind 例（背景・文字・ボーダー） |
|---|------|--------|--------------------------------------|
| 1 | 介入スタイル | 赤 | red（bg-red-50, text-red-700, border-red-200 等） |
| 2 | 知覚対象 | 緑 | emerald（bg-emerald-50, text-emerald-700, border-emerald-200 等） |
| 3 | 判断基準 | 黄 | amber（bg-amber-50, text-amber-700, border-amber-200 等） |
| 4 | 場の関わり | 青 | sky（bg-sky-50, text-sky-700, border-sky-200 等） |

**補足**

- 2.8.1の「タイプ名を図の上または下に表示」は、中央に重ねないことで重なりを解消する方針。図の直上キャプションまたは直下キャプションとしてタイプ名を表示する形でよい。
- 2.8.2の「文字の大きさを工夫」は、軸名をやや大きく・太く、型ラベルをその次、強み4語を本文相当のサイズにするなど、段階的な階層を設けることを想定する。

### 2.9 結果画面の表示改善（4象限・強み・相性）（2.9 要件）

**4象限図**

- **タイトル**: 図の上の見出しは「**あなたのファシリテーション特性**」とする（小さく表示）。
- **サブタイトル**: 「〇〇型ファシリテーター × △△グループ」を表示（触発/見守型 × 推進者/共感者/戦略家/守護者グループの4パターン）。
- **リード文**: サブタイトル直下に、型＋グループの説明を組み合わせた1文を表示する。
- **表内の構成**: 各象限で**軸名**と**型ラベル**は改行せず1行で表示する。形式は「軸名：**型ラベル**」（例：介入スタイル：**見守型**）。型ラベルは太字で強調する。
- **強み部分**: 各象限のキーワード（強み4語）は**箇条書き**とし、先頭に「・」を付けて表示する。

**強みセクション**

- 結果画面の「強み」4カードは**1色（エメラルド系）**で統一する。軸別の色分け（2.8.3）は**強みセクションには適用しない**。トップの4軸説明・傾向キーワード図・ファシリテーター特性（スコアバー）のみ軸色を適用する。

**他のスタイルとの相性**

- 「相性が良いスタイル」「難しいかもという相性」それぞれに複数スタイル（2スタイル等）がある場合、**読み手が「このスタイルがある」と一目で分かる**ようにする。
- 各スタイルを**フォントの強弱**（スタイル名を太字・やや大きく）や**区分け**（スタイルごとにカード・枠で区切る）で明確に分離し、2つあることが視認できるようにする。

## 3. 機能要件

### 3.1 画面構成

```
[トップページ] → [質問画面(32問)] → [結果画面] → [シェア/保存]
```

### 3.2 各画面の詳細

#### トップページ

**ヒーローセクション**
- awareness=designロゴ画像
- アプリ名「ファシリスタイル」
- サブタイトル「ファシリテーター特性診断」
- アプリの説明（何が分かるか）
- **診断で測定する4つの軸**（カード形式で表示）。各軸に補足文言（何を測るか）を1行追加。軸色は2.8.3に準拠：介入スタイル＝赤、知覚対象＝緑、判断基準＝黄、場の関わり＝青。※4象限図（4つのスタイル）はTOPから削除済み（2026-01-25）。16タイプの詳細説明内にスタイル文脈は残す。
- 所要時間の目安（約5分）
- 「診断を始める」ボタン
- **注釈**（ボタンの下に配置）
  - 1) 「スタイルはあくまで得意としている傾向であり、どの視点もバランスよく持つことが重要だと考えています。」（中央寄せ・太字・text-xs）
  - 2) 「※この診断は自己理解のためのツールです。結果は参考情報としてご活用ください。」
  - 16スタイル一覧ブロック下の「もう一度診断」ボタン直下にも同様の2文を表示
- 「あなたのファシリテーションスタイルを発見しよう」「診断で測定する4つの軸」「32問で以下の4つの軸を測定します。これらの組み合わせから16スタイルに判定されます。」は中央寄せ

**ファシリテーター16スタイル一覧セクション**
- 「ファシリテーター16スタイル」セクション
- 触発型ファシリテーター
  - 推進者グループ（4タイプ）… 深い赤（red-900）
  - 共感者グループ（4タイプ）… 深い赤・薄め（red-800）
- 見守型ファシリテーター
  - 戦略家グループ（4タイプ）… 参照青（#1F86C8）
  - 守護者グループ（4タイプ）… 参照青・薄め（#38a5d8）
- **4グループ色ルール（案D改3）**：塊の色の同系統で2グループ濃淡。触発型塊=深い赤系統（推進者=red-900、共感者=red-800。カード背景 red-50。ビビットにせず落ち着いたトーン）、見守型塊=添付参照の青系統（#1F86C8 / #F0F8FF）。
- 各スタイルをカード形式で表示（スタイル名・ニックネーム、キャッチコピー）。カードクリックで詳細モーダル表示。モーダル内「ファシリテーション特性」も同様に見出し太字・サブタイトルは塊と同色。

#### 質問画面

- プログレスバー（進捗表示）
- 質問番号（例：質問 1 / 32）
- 質問文
- 選択肢の両極（AとBのラベル）
- 6段階の回答ボタン（1〜6）

#### 結果画面

**セクション1: あなたのスタイル**（採用: 1-A）

- 「あなたのファシリテータースタイルは...」→ **スタイル名（ニックネーム）** を大きく表示（例：「場の指揮者」）
- キャッチコピー
- **4軸タグを1行で表示**（例: （見守型）（洞察型）（関係型）（即興型））。診断結果の4軸傾向ラベルを括弧付きで並べ、何型かが一目で分かるようにする
- タイプの詳細説明（2〜3段落）
- **傾向キーワード（あなたのファシリテーション特性）**（2.7・2.9）: タイプ詳細説明の下、「得意な場面」の直前に表示する。構成は 1) タイトル「あなたのファシリテーション特性」 2) サブタイトル「〇〇型 × △△グループ」 3) リード文（型＋グループの説明） 4) **4象限**（左上＝介入スタイル、左下＝判断基準、右上＝知覚対象、右下＝場の関わり）。各セルで「**軸名：型ラベル**」を1行で表示し、強み4語は「・」付き箇条書き。角Rなしの表形式。軸色は2.8.3に準拠。
- 得意な場面（タグ形式で表示）
- フェードインアニメーション

**セクション2: ファシリテーター特性**（採用: 2-D）

- セクション名は**「性格特性」ではなく「ファシリテーター特性」**とする
- **特性の構成要素**（各軸ブロック）の構成・フォントサイズは、添付画像（16 Personalities風）に準じる
  - 現状、タイトル（例：50％見守型）とリード文（例：参加者の内省や対話を邪魔せず…）が離れて読みづらいため、**タイトルとリード文を近づけて一まとまりで読めるようにする**
  - 構成: 上に**スコアバー**（フル幅・中央線・○インジケーター）、その**直下**に**軸名: XX% 〇〇型**を1行で表示（％は大きく・太字・アクセント色）、さらに**直下**に**リード文**（各軸の簡易説明）を配置し、タイトルとリード文の間の余白を小さくする
  - フォントサイズ感: 軸名・％・型名は大きく、リード文はやや小さめで、添付のような階層にする
- 4軸のスコアをフル幅のバーで可視化。中央に縦線を入れ、○の位置がスコアと一致する
- **表示％のロジック**: バーの**中心（スコア28）を0%**とし、「該当の型に○%寄っている」を表示する。左寄りなら左端（触発型など）に、右寄りなら右端（見守型など）に、中心から何%寄っているかを0〜100%で示す。中央付近は0%、端に近いほど100%に近づく。

**セクション3: 強み・注意点**

- 2カラムレイアウト（強み / 注意点）。**強み**4カードは1色（エメラルド系）で統一（2.9）。
- 各軸から代表的な項目を抽出してカード形式で表示
- 各項目にタイトルと説明文

**セクション4: ファシリテーターとしての影響力**（採用: 3-B）

- 「各軸の詳細分析」は廃止し、**「ファシリテーターとしての影響力」**セクションを1つだけ設ける
- 16タイプごとに「場や参加者にあなたが与える影響」を表す**1本の文章**を用意し、診断結果タイプに紐づけて表示
- **リード文は400文字程度まで長くしてよい**。読み手がわかりやすく、かつ**エンパワメントする**ような文章にする
- 特性を踏まえたエンパワメントする文章で表現する

**セクション5: コーファシリとの組み合わせヒント**（採用: 4-A）

- タブ切り替え（メイン / サブ）は維持
- **中身は軸別4ブロックではなく、診断結果タイプ用の「1本の文章」**で表示
  - メイン時: 「このようにサブファシリと振る舞おう！」のような文章
  - サブ時: 「メインファシリとこのように振る舞うと良い」のような文章
- **各タブのリード文は400文字程度まで長くしてよい**。読み手がわかりやすく、かつ**エンパワメントする**ような文章にする
- 16タイプ×2（メイン時・サブ時）の計32文を用意し、`result.type.id` で参照
- **「コーファシリテーションのポイント」**はコーファシリの文脈として、メイン／サブそれぞれのタブ内容の中に表示する（事前に役割分担を明確にする、交代のタイミングや合図を決めておく、ワークショップ後に振り返りの時間を確保する）

**セクション6: 他のスタイルとの相性**（採用: 4-C・独立項目・2.9）

- **「コーファシリとの組み合わせヒント」とは別の項目**として、結果画面に独立したセクションで表示する
- 相性が良いスタイル（2〜3スタイル）＋それぞれの振る舞いヒント（1行）
- 相性が難しいスタイル（2〜3スタイル）＋対処する振る舞いのヒント（1行）
- **視認性**: 複数スタイルがある場合、スタイル名を太字・やや大きくし、スタイルごとにカードや枠で区切るなど、読み手が「このスタイルがある」と分かるようにする。

**セクション7: シェア・保存**

- SNSシェアボタン（X/Twitter, Facebook, LINE）
- 結果画像ダウンロードボタン
- **SNS投稿内容の要件定義**: 投稿文・結果サマリ・画像・awareness=design 告知・拡散方針の詳細は [09_sns_share_requirements.md](09_sns_share_requirements.md) を参照

## 4. 非機能要件

### 4.1 プラットフォーム

- Webアプリ（SPA）
- スマートフォン対応（レスポンシブデザイン）

### 4.2 デザイン方針

全体のデザインは **FacilitatorStyles ロゴ**（丸み・柔らかさ・ライトブルー＋グレー）に合わせて統一する。詳細は [13_design_direction.md](13_design_direction.md) を参照。以下は 2026-01-25 時点の実装済み方針。

- **方針の軸**: ロゴの「丸み・流れ・親しみやすさ」に合わせ、角を抑え・余白を活かしたクリーンなUI
- **スタイル**: ミニマル・フラット、やわらかい印象（ロゴ連動）
- **余白**: 多めに取ったクリーンなUI（セクション間 py-16〜24、コンポーネント間ゆったり）
- **タイポグラフィ（実装済み）**:
  - **フォント**: **M PLUS Rounded 1c**（丸ゴシック）を採用。Google Fonts で 400 / 500 / 700 を読み込み
  - 本文: 行間 1.75、見出しは font-bold（700）、letter-spacing 広め
- **カラーパレット（ロゴ連動・実装済み）**:
  - **プライマリ**: **sky 系**（Tailwind primary を sky パレットで定義）。ボタン・結果ヒーロー・プログレスバー・モーダルヘッダー等に使用（primary-600〜700）
  - **ニュートラル**: **slate 系**。本文・サブテキスト・ボーダー・背景（gray は使用せず slate に統一）
  - **背景**: 白（メイン）、slate-50（サブエリア例: 16スタイル一覧セクション）。コーファシリのポイント枠は sky-50
  - **4軸・4グループ色**: 2.8.3 および案D改3の通り維持（介入＝赤、知覚＝emerald、判断＝amber、場＝sky。触発型＝深赤、見守型＝参照青 #1F86C8 / #38a5d8）
- **形（実装済み）**: 角丸を統一。カード rounded-2xl、ボタン rounded-full、チップ・入力・カード内要素は rounded-xl 以上。ボーダーは 1px・slate 系の薄いトーン。シャドウは shadow-sm〜shadow-md の柔らかい範囲
- **アニメーション（実装済み）**: フェードイン（fadeIn, fadeInUp）、トランジション 300ms ease-out。ホバーは **scale-[1.02]** に抑えめ（バウンスや派手なイージングは使わない）
- **カード**: 角丸 2xl、border-slate-100、ホバーで border-slate-200・軽いシャドウ
- **ボタン**: 角丸フル（rounded-full）、primary-600、ホバーで primary-700・scale-[1.02]・shadow-md

### 4.3 技術スタック

- **フロントエンド**: React 18 + Vite 5 + TypeScript
- **スタイリング**: Tailwind CSS 3
- **画像生成**: html2canvas（結果画像保存用）
- **ホスティング**: Vercel
- **アクセス解析**: Google Analytics（want要件）
