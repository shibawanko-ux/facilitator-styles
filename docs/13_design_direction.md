# デザイン方針（ロゴ連動）

FacilitatorStyles ロゴのトーンに合わせた全体のデザイン方向性を定義する。ロゴを「デザインの軸」として、タイポグラフィ・色・形・トーンを揃える。

---

## 1. ロゴの特徴（デザインの軸）

- **書体**: ボールドで丸みを帯びた、ややプレイフルなディスプレイ系
- **印象**: 角がなく、柔らかく流れるような見た目
- **色**: 「Facilitator」＝ライトブルー／スカイブルー、「Styles」＝ミディアムグレー
- **トーン**: 親しみやすく、現代的。堅すぎず、かといって軽すぎない

→ 全体も「丸み・柔らかさ・親しみやすさ」をベースにしつつ、読みやすさと信頼感を保つ。

---

## 2. デザインの方向性

### 2.1 トーン＆マナー

- **キーワード**: やわらかい、丸い、親しみやすい、クリーン、余白を活かす
- **避ける**: 角ばった幾何学だけのデザイン、無機質なコーポレート風、ごちゃごちゃしたレイアウト
- **ロゴとの関係**: ロゴが「柔らかく丸い」ので、UIも角を抑え、丸み（角丸・曲線）を一貫して使う。

### 2.2 タイポグラフィ

- **方針**: ロゴの「丸み・流れ」に合わせ、できるだけ角張らないフォントを選ぶ。
- **推奨**:
  - **本文・UI**: **M PLUS Rounded 1c**（丸ゴシック）  
    - ロゴの丸さと相性が良く、可読性も確保できる。
  - **代替**: 現状の **Noto Sans JP** を継続する場合は、字間をやや広め・行間もゆったりにして「硬さ」を和らげる。
- **ルール**:
  - 見出しは太め（700）で、余白を多めに。
  - 本文は行間 1.7〜1.8 程度で読みやすく。
  - 文字間隔は広め（tracking や letter-spacing で調整可）。

### 2.3 カラーパレット（ロゴ連動）

- **ベースカラー（ロゴに合わせる）**
  - **ライトブルー**: ロゴ「Facilitator」の色。メインのアクセント・リンク・重要要素に使用。  
    - 例: Tailwind `sky-500`〜`sky-600`、またはロゴに近いカスタム色。
  - **ミディアムグレー**: ロゴ「Styles」の色。本文・サブテキスト・ボーダーに使用。  
    - 例: `slate-500`〜`slate-700`。背景は `slate-50` など薄いグレーで統一。
- **セマンティックカラー（診断UIで既存の意味を維持）**
  - 触発型: 深い赤（red-800〜900）
  - 見守型: 参照青（#1F86C8, #38a5d8）
  - 4軸: 介入＝赤、知覚＝emerald、判断＝amber、場＝sky  
  - これらはそのまま使い、ベースの「ライトブルー＋グレー」とバランスを取る。
- **背景**
  - メイン: 白（`white`）
  - サブ: ごく薄いグレー／スカイ系（`slate-50` または `sky-50`）で、ロゴのライトブルーとつながりを持たせる。
- **結果画面ヒーロー**
  - 「あなたのファシリテータースタイルは...」の背景は**グレー（slate-600）**で統一。テキストは白・slate-200。PC・スマホ共通。
- **回答スケールのA/B両極（回答画面専用）**
  - **A（質問の左側・第一の極）**: 赤系統（`red`）。「A側」であることを一貫して示し、回答スケールの左側（1〜3）と対応する。
    - **Aバッジ（ラベル）**: `bg-red-100 text-red-800 border-red-300`。サイズは `px-3 py-1`、`text-xs`。
    - **A設問テキスト**: `text-red-800 font-bold` で該当カラー・太字にして見やすくする。
    - **◯選択時（1〜3のいずれかを選択）**: `bg-red-600 text-white`、`ring-red-400` で該当の◯に色をつける。
  - **B（質問の右側・第二の極）**: 青系統（`blue`）。「B側」であることを一貫して示し、回答スケールの右側（4〜6）と対応する。
    - **Bバッジ（ラベル）**: `bg-blue-100 text-blue-800 border-blue-300`。サイズは `px-3 py-1`、`text-xs`。
    - **B設問テキスト**: `text-blue-800 font-bold` で該当カラー・太字にして見やすくする。
    - **◯選択時（4〜6のいずれかを選択）**: `bg-blue-600 text-white`、`ring-blue-400` で該当の◯に色をつける。
  - **視認性**: 背景と文字のコントラストを十分に取り、ラベルとして識別しやすくする。回答UIのA/B専用ルールとする。
  - **レイアウト**: A・Bのアイコンと文字は**中央寄せ**。A/Bブロックの**間に隙間**（gap-6〜8）を入れ、左右の余白を抑えつつ中央の境目でエリアが分かるようにする。

### 2.4 形・コンポーネント

- **角丸を統一**: ロゴに「角がない」ので、UIも角を立てない。
  - カード: `rounded-2xl`（大きめの角丸）
  - ボタン: `rounded-full` または `rounded-xl`
  - 入力・チップ: `rounded-xl` 以上
  - 小さなバッジも `rounded-full` または十分な角丸
- **シャドウ**: 強く硬い影は避け、`shadow-sm`〜`shadow-md` の柔らかい影に留める。
- **ボーダー**: 太く鋭くしない。`border` は細め（1px）で、色はグレー系の薄いトーン。

### 2.5 余白・レイアウト

- **余白多め**: ロゴ周りも含め、要素間はゆったり取る。
- **流れ**: 縦のリズムを一定にし、セクション間は `section` や `py-12` などでまとまりを作る。
- **中央寄せ**: ヒーロー・キャッチコピー・主要メッセージは中央寄せで「ひとまとまり」に見せる（現状のTOP方針と一致）。

### 2.6 アニメーション

- **控えめで滑らか**: ロゴの「流れ」に合わせ、動きは少なめ・遅め・イージングは ease-out 系。
- **既存維持**: フェードイン・フェードインアップ・ホバー時の軽いスケールはそのまま活用。
- **避ける**: バウンスや派手なイージングは使わない。

---

## 3. 実装時のチェックリスト

- [x] フォント: M PLUS Rounded 1c の導入、または Noto Sans JP の余白・字間で丸み寄せ
- [x] ベース色: プライマリ／ニュートラルに sky 系＋slate 系を採用
- [x] 角丸: カード・ボタン・チップが rounded-xl 以上で統一されているか
- [x] 余白: セクション間・コンポーネント間が十分か
- [x] ロゴ周り: ヒーローでロゴが主役になり、余白とバランスが取れているか
- [x] 既存の4軸・4グループ色は維持しつつ、背景・テキスト色がロゴのグレー／ブルーと調和しているか
