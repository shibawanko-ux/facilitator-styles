# ブログ作成アシスト（Cursor のみ）

note・Goodpatchブログ・Works 等に載せる記事を、**Cursor のチャットだけで**作成します。ターミナルや別アプリは使いません。

## 使い方

1. **このプロジェクトをワークスペースとして開く**（`Project/app_blog_requirements/` を開く、または親フォルダからこのフォルダを含むワークスペースを開く）。
2. **ブログ記事作成用の Cursor ルールが有効なチャット**で、次のように送る：
   - 「ブログ記事を書きたい」
   - 「記事を書きたい」
3. AI が **どの媒体で書くか**（Note / Goodpatchブログ / Works / Works風ブログ）を聞くので、1つ選ぶ。
4. 続けて **記事サブモード** を選ぶ：
   - **1. 一緒にヒアリングで詰める** — 質問に答えていき、テーマ・読者・主張・文体などを決めて定義書をまとめる。
   - **2. テキストから構成する** — 発話録・要約・本の抜粋などを貼り付け、そこから定義書を組み立てる。
5. 定義書がチャット内でまとまったら、**そのまま同じチャットで** 構成案（3案）→ 選択 or 追加提案 → 記事要約 → 要約修正 → 本文（約5000字）→ 微修正 → タイトル確認 → 完成・保存、まで進める。

既にブログ記事要件定義書のファイルがある場合は、そのファイルを **@ で指定**して「この定義書で構成案を出して」と送れば、構成案から進められます。

## 記事ごとの成果物の格納（推奨）

毎回の記事で出力した定義書・構成案・要約・本文を残す場合は、**`articles/`** の下に **1記事1フォルダ** を用意することを推奨します。

- **フォルダ名の例**: `articles/2025-02-21_心理的安全性/`
- **中に保存するファイルの例**:
  - `01_要件定義書.md` — ブログ記事要件定義書
  - `02_構成案.md` — 採用した構成案（任意）
  - `03_記事要約.md` — タイトル・各章タイトル・リード文
  - `04_本文.md` — 完成した記事本文

保存は、チャットの内容をコピーしてファイルに貼るか、完成時に「この内容を `articles/2025-02-21_〇〇/04_本文.md` に保存して」と Cursor に依頼してファイルを生成させてください。

## サブモードの違い

| モード | 概要 |
|--------|------|
| **1. ヒアリングで詰める** | テーマ・想定読者・主張・素材・目的・文体・（該当時）Awareness designの扱い・文字数などを順に質問し、最後に定義書をチャット内で出力。 |
| **2. テキストから構成する** | 参考テキストを貼り付けると、AI がヒアリング項目を埋めた定義書を返す。必要なら編集して送り直し、変更確認のあと構成案を複数案提示してから本文へ。 |
