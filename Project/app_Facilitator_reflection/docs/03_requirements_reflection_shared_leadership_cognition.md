# 振り返りの拡張要件（シェアドリーダーシップ・知覚スキーマ認知）

本ドキュメントは、[01_requirements.md](01_requirements.md) の 2.4 結果の表示 および 2.4.1 分析結果画面の拡張 における「振り返り用の対話の問い」を**拡張する**要件を定義する。シェアドリーダーシップの振り返りと、組織行動論（知覚・スキーマ・認知）の考え方を振り返り対話に組み込むことで、**関係性の深化**と**行動変容**という本アプリの目的に沿った振り返りを実現する。

---

## 0. 目的と位置づけ

- 本ドキュメントは 01 の 2.4 / 2.4.1 の**振り返り部分**を拡張する。
- アプリの目的「**問いを通じた関係性の深化と行動変容の支援**」（01 の 1.2）との整合を前提とする。
- 既存の「振り返り用の対話の問い」（3問）に、以下 1・2 の観点を反映した問いを**追加**する。

---

## 1. シェアドリーダーシップの振り返り

### 1.1 目的

- メインとサブの**関係性**と、**二人でリーダーシップを取っていた瞬間**を可視化する。
- 振り返りを通じて、メイン個人の評価に偏らず、**ペア（チーム）としての振る舞い**を言語化し、次回に活かす。

### 1.2 達成したいこと

- **「いつ」の特定**: どの場面（オープニング、議論の山場、まとめ、時間押しのとき など）で、二人でリーダーシップを取っていたか／取れていたと感じたかを特定する。
- **役割の受け渡し・分担の言語化**: 主導の受け渡し、サブの補い、メインの委譲、役割分担がどうだったかを行動レベルで語れるようにする。
- **相互に「あの時、リーダーシップ発揮してたね」と言い合う**: 評価ではなく、**相手のリーダーシップを発揮していた具体的な瞬間**を名前で伝え合い、承認し合うことを促す。

### 1.3 要件

| ID | 要件 | 受入条件 |
|----|------|----------|
| R-SL-1 | 振り返りの問いの中に、**メイン・サブのリーダーシップを相互に言語化する**問いが含まれる | 相手のリーダーシップを発揮していた瞬間を具体的に挙げて伝えることを促す問いが、表示される振り返りに含まれる |
| R-SL-2 | 問いで、**「いつ」（場面）** を特定することを促す | 特定のタイミング・場面を挙げて振り返ることを促す文言または意図が含まれる |
| R-SL-3 | 問いで、**二人で試したいこと**（受け渡し・サイン・フォローなど）を決めることを促す | 次のファシリテーションでメイン・サブのペアとして試すアクションを対話で決めることを促す問いが含まれる（既存の「メインが1つ試す」と並立可能） |

## 2. 組織行動論の活用（知覚・スキーマ・認知）

### 2.1 用語の定義

振り返りで共通言語として使うため、次のように定義する（組織行動論における一般的な考え方に基づく）。

- **知覚（perception）**: 同じ場面でも、**何に注目し、どう解釈するか**は人・役割によって異なる。振り返りでは「あの場面で自分は何を見ていたか」を共有することを促す。
- **スキーマ（schema）**: 「良いファシリテーション」「うまい介入」など、各自が持つ**心の枠組み**。振り返りでは「うまくいったと感じた理由」と「他者が違う見方をしていたとしたら、どんな枠の違いがありそうか」を扱う。
- **認知（cognition）**: 状況の**意味づけ・判断・記憶**の仕方が人によって違う。振り返りでは「なぜそう解釈したか」「何を根拠にそう思ったか」を短く共有することを促す。

### 2.2 振り返りでの活用目的

- 見え方の違いを**個人攻撃ではなく、知覚・スキーマ・認知の違い**として扱う。
- スコアやコメントの食い違いを「誰が悪い」ではなく「**認知の違い**」として検討できるようにする。
- メイン・サブ・参加者それぞれの「意味づけ」を並べることで、**多様な解釈**を学びの材料にする。

### 2.3 要件

| ID | 要件 | 受入条件 |
|----|------|----------|
| R-COG-1 | 問いで、**その場面で何に注目したか**（知覚）を共有することを促す | あの場面で何に一番注目していたかを一言で共有することを促す問いまたは意図が含まれる |
| R-COG-2 | 問いで、**解釈の違い**（スキーマ・認知）を扱うことを促す | 「うまくいった」と感じた理由や、他者と解釈が違っていた場合の違いから学ぶことを促す問いが含まれる |

## 3. 振り返りの問いの拡張

### 3.1 追加方針

- 既存の「振り返り用の対話の問い」（3問）に、上記 1・2 を反映した問いを**追加**する。
- 追加は **問いの数・ブロック構成** のいずれかで行う（セクション 4 で選択肢 A・B を定義）。
- 各問には **意図（intent）** を定義する。固定文言は実装フェーズで整備する。

### 3.2 問いの例（案）

以下は実装時に文言を整備するための例である。

| 観点 | 問いの例 | 意図の例 |
|------|----------|----------|
| シェアドリーダーシップ | 今回の場で、メインとサブが「二人でリーダーシップを取っていた」と感じた瞬間はいつでしたか？ そのとき、主導の受け渡しや役割分担はどうなっていましたか？ | シェアドリーダーシップが発揮された場面の特定と、行動の言語化。 |
| シェアドリーダーシップ（相互承認） | 相手（メイン／サブ）の「リーダーシップを発揮していた」と感じた具体的な瞬間を、ひとつ挙げて伝えてください。 | 「あの時、リーダーシップ発揮してたね」と言い合うことを促す。 |
| 知覚・認知 | あの場面で、あなたは何に一番注目していましたか？ 同じ瞬間を、メイン・サブ・参加者でどう解釈していたか、一言ずつ共有してみましょう。 | 知覚・解釈の違いを可視化し、認知の違いとして学びの材料にする。 |
| 二人で試す | 次のファシリテーションで、メインとサブの「二人のリーダーシップ」として、1つだけ試してみたいことは何ですか？ | ペアとしての次の一手を対話で決める。 |

## 4. 表示・データの要件

### 4.1 表示方針の選択肢

表示は **A または B のいずれか** で実装する。どちらでも本要件の目的（R-SL-1〜R-SL-3、R-COG-1〜R-COG-2）を満たす。

| 選択肢 | 説明 | 備考 |
|--------|------|------|
| **A. 問いの統合** | 既存の 3 問に 2〜3 問を追加し、`reflectionQuestions` を 5〜6 問として一覧表示する。 | 実装時は 09 の出力フォーマット・件数制約の更新、バリデーションの更新が必要。 |
| **B. 別ブロック** | 既存「振り返りの問い」は 3 問のまま。新ブロック「シェアドリーダーシップ・認知の振り返り」を設け、そこに 2〜3 問を表示する。 | 実装時はレポート出力に新キー（例: `sharedLeadershipReflectionQuestions`）の追加、結果画面に折りたたみ可能な新セクションの追加が必要。 |

### 4.2 サブがいないルームでの扱い

- サブファシリテーターが存在しないルーム（メイン＋参加者のみ）では、**シェアドリーダーシップに特化した問い**は表示しない、または省略することを推奨する。
- 知覚・スキーマ・認知の問いは、メインと参加者との間でも有効なため、**表示してよい**。

### 4.3 レポートエージェントとの整合

- 振り返りの問いの拡張は [11_report_agent.md](11_report_agent.md) の出力構造に影響する。選択肢 A の場合は `reflectionQuestions` の件数拡張、選択肢 B の場合は新キーの追加で対応する。具体的な JSON 変更は実装フェーズで 09 を更新する。
