# Part A：6つのコアスキル（講義要約）

## A1. 6コアスキルと既存評価項目の対応

| 講義の6コアスキル | 講義での定義（要約） | 本アプリでの対応項目 |
|-------------------|----------------------|------------------------|
| **1. 説明力** | 焦点・背景・意図・前後のつながりを伝える。不足するとテーマに引きつけられない。 | **E3** 目的・進め方・問いが参加者に伝わっていたか？ |
| **2. 場の観察力** | 事実を収集し、場の状態を解釈・見立てる。心理や関係性を推測し、介入の要否を判断。 | **B4** 参加者の反応や空気を捉え、場に活かしていたか？ |
| **3. 即興力（インプロビゼーション）** | 台本に縛られず、Yes and で柔軟に振る舞い、進め方・問いを微修正。 | **F3** 状況に合わせて柔軟に進め方や問いを変えられていたか？ |
| **4. 情報編集力** | 複数情報を組み合わせて新しい意味にまとめる。共通点・対立点の構造化、不足視点の探索。 | **E4** 発言や成果が全体で整理され、見える化されていたか？／**F4** 時間と内容のバランスが取れ、流れが編集されていたか？ |
| **5. リフレーミング力** | 枠組みを変えて捉え直し、場の意味づけを変える。ブレイクスルーの起点。 | **B5** 発言を違う角度で返し、気づきや深まりを促せていたか？ |
| **6. 場のホールド力** | 安心して創造性を発揮できる状況を作りつつ、目標達成に責任を持つ。黒子的な支え。 | **B6** 安心して話せる・挑戦できる場が保たれていたか？ |

## A2. 行動の流れと3視点の関係

- **準備**: 説明力
- **即興**: 場の観察力、即興力、情報編集力、リフレーミング力
- **流れ**: ①説明力（事前準備）→ ②場の観察力（インプット）→ ③判断 → ④即興力・情報編集力・リフレーミング力（アウトプット）→ **場のホールド力**が結果として生まれる

**評価項目の3視点（鷹の目・虫の目・魚の目）**との対応（入力画面の項目グルーピング。集計・結果画面の表示は**6軸**で [14_report_structure.md](14_report_structure.md) を参照）:

- **鷹の目（全体）**: 説明・編集・流れの「設計と見える化」（E1–E4）
- **虫の目（参加者）**: 観察・リフレーム・ホールド・問いかけ（B1–B10）
- **魚の目（流れ）**: 即興・編集・時間と内容のバランス（F1–F4）

## A3. テーブルファシリテーションとの関係

- **創発系（商品開発）**: ロールモデルとして自らも発想、キーマンの「下ごしらえ」
- **学習系（人材育成）**: フィードバック・総括・ミニレクチャー、メインの説明が届いていないときの「助け船」
- **対話・内省**: 傾聴と問いかけのロールモデル、問いかけのリレーが始まったら介入を減らす

本アプリでは **E1** に「テーブルファシリの行動含む」と含めています。

# Part B：Facilitation Design（WS資料）

## B1. ファシリテーションの定義（WS資料より）

- **大辞林**: グループによる活動が円滑に行われるように支援すること。組織が目標を達成するために、問題解決・合意形成・学習などを支援し促進すること。
- **語源**: Facilis（容易にする・円滑にする・助長する）→ 集団による知的相互作用を促進する働き。
- **狭義**: 会議やワークショップの場として前に立ち、参加者に問いを投げながら、創造的対話のプロセスを支援する行為。
- **広義**: 問題の本質を捉え直し、解くべき課題を定義し、問題解決のプロセスに伴走すること。
- **ファシリテーター**: 中立の立場でプロセスに着目し、ゴールを達成するため、人々の能力を最大限引き出す役割。

## B2. Facilitation Design の5要素と3視点の対応

| 要素 | 内容（WS資料より） | 本アプリの視点 |
|------|---------------------|----------------|
| 場を見立てて設計する | 目的・目標設定、参加者の定義、場のレイアウト、発散・収束の設計、時間配分 | **鷹の目**（全体設計・説明・見える化） |
| 場を作り、つなげる | 目的やゴールの合意、話しやすい雰囲気、進め方の設計、傾聴で安心感・信頼感 | **虫の目**（参加者・安心感・問いかけ） |
| 受け止めて引き出す・まとめて分かち合う | 傾聴、質問で意見を引き出す、柔らかく主張して方向づける、曖昧な主張の明確化、議論の図解 | **虫の目＋魚の目**（観察・編集・流れ） |
| 噛み合わせて、構造化する | 全体像の把握、多様な視点から議論・構造化、対立をチャンスと捉える、言葉の奥の本音を探る | **鷹の目・魚の目**（情報編集力・流れの編集） |
| 振り返り・次に向けて行動 | 目的は達成したか、参加者に変化はあったか、アクションできるか、アウトカムはどうか | **結果画面**の振り返りの問い・アクション提案 |

→ **共有・発散・収束・決定**のプロセスは、評価項目の「場のデザイン」「構造化」「合意形成」と直結しています。

## B3. 場のデザインと評価項目

- **場の意味**: 議論で変化させたいもの・ゴール状態の共有。→ **E2**, **E3**
- **場の状態**: 参加者3階層（A/B/C）、アイスブレイクで雰囲気作り。→ **B6**, **B2**
- **場の環境**: 議論の地上戦（WB・スクライブで可視化）、発言者と内容を切り離す。→ **E4**

## B4. 対人関係デザイン（傾聴・問い）と評価項目

- **傾聴**: 傾聴3つのキク（聞く・聴く・効く）、スタンス10。→ **B4**, **B6**, **B7〜B10**
- **問いのスキル**: 溝に気づく→向こうを眺める→渡り橋を設計→問いかけて場を動かす。→ **B5**, **B7〜B10**
- **ファシリテーターと参加者の関係**: 問いで全員から主張を引き出す、1つの意見を触媒にして全員で議論。→ **B1**, **B2**, **E1**

## B5. 構造化デザイン（スクライブ・板書）と評価項目

- **スクライブ（板書）**: キーワードを拾い要約→装飾・枠で強調→囲みや線・矢印で関係表現→図解で体系的に整理。→ **E4**, **F4**
- **心理に寄り添う振る舞い**: 言葉の奥の本音を探る。→ **B5**, **B7**

## B6. 全体設計・合意形成と評価項目

- **全体設計**: 目的・ゴール、参加者、進め方（Agenda・内容・時間）、場の設備、制約条件。→ **E3**, **F1**, **F2**
- **合意形成**: コンセンサス、対立は悪者ではない、氷山モデル。→ **E4**, **B5**, **F3**
- **スクリーマー（肯定性を上げる3ステップ）**: 正しい要素を見つける→意図を再提示する→方向転換を促す。→ 振り返りの問い・アクション提案に活かせる。

## B7. 振り返り（WS資料）の文言

- 「目的は達成したか？」「参加者に変化はあったか？」「アクションできるか？」「アウトカムはどうか？」
- 「今回のワークで『仕事に利用できそうなポイント』が可視化されている状態」をゴールにした振り返りAgenda。
- → 結果画面の「振り返りの問い」「アクション提案」に織り込めます。

# 活用の具体案（評価フォーム・結果）

1. **評価入力画面の「観点の説明」**  
   共有・発散・収束・決定、Facilitation Design の5要素、鷹・虫・魚の対応を短く表示（※実装済み）。

2. **項目ラベルやツールチップ**  
   E3 に「説明力：何をするか・どうやるか・なぜやるか」、E4 に「スクライブ（板書）で議論が見える化」など、講義・WSの用語を括弧で補足可能。

3. **結果画面の振り返り**  
   「目的は達成したか？」「参加者に変化はあったか？」を共通文に。reflectionByType の文言を「準備／即興」「場のホールド」「ロールモデル」と整合。

4. **05_evaluation_items.md**  
   参照元に本リファレンス（10_reference_facilitation.md）を明記し、主要項目に講義・WSの用語を要約で記載。
