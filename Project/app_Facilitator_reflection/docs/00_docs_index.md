## 初めて読む人：どこから読むか

- **「何のアプリか・何ができるか」を知りたい**  
  → **01_requirements.md** から。

- **「手元で動かしたい」**  
  → **09_startup_guide.md** に従い、必要なら **.env.example** をコピーして `.env` を用意。

- **「実装や改修をステップで進めたい」**  
  → **08_implementation_guide_beginner.md** の Step 1 から。Supabase や DB の前提は **04_data_design.md** を参照。

- **「評価項目や観点の根拠を知りたい」**  
  → **05_evaluation_items.md** と **10_reference_facilitation.md**。

- **「本番デプロイやセキュリティを確認したい」**  
  → **06_security.md** と **09_startup_guide.md**（本番手順は 09 に追記予定）。

- **「チャットごとに役割を分けて作業したい」**  
  → **16_cursor_chat_roles.md** で役割を選び、コピー用ブロックをチャットの最初に貼り付ける。

## 更新のしかた

- 要件・スコープの変更 → 01, 07 を更新。
- テーブル・RLS の変更 → 04, 06 を更新。
- 評価項目の追加・変更 → 05 を更新し、必要なら 10 にも反映。
- 新しい実装ステップ → 08 に追記。
- 起動・デプロイ手順の変更 → 09 を更新。
- 新しい講義・WS 資料などのナレッジ → **10_reference_facilitation.md にのみ**追記（運用方針）。
- 品質ゲート・AI 整形の変更 → **13_quality_gate_and_polish.md** を更新。
- 結果画面・全画面のデザイン方針の変更 → **15_design_direction.md** を更新。
- チャット役割の追加・変更 → **16_cursor_chat_roles.md** を更新。

このファイル（00_docs_index.md）は、docs の構成が変わったときにだけ更新してください。
