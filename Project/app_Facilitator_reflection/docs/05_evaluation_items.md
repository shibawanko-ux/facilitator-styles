# ファシリテーターリフレクション 評価項目一覧

## 1. 概要

- 評価入力画面で使用する項目を、**鷹の目・虫の目・魚の目**の3視点で構成する（項目IDは E1〜F4 の18項目のまま）
- **集計・表示では6軸（6コアスキル対応）にグルーピング**し、レーダーチャート・分析コメントは6軸の総合点を用いる
- MIMIGURI の [CULTIBASE](https://www.cultibase.jp/) に掲載の**ファシリテーションの6つのコアスキル**および**問いかけの基本定石**を参照して項目を設計している
- 各項目はスコア（例: 5段階）＋ 自由記述で回答する

## 4. 6軸と項目の対応（集計・レーダー用）

結果画面のレーダーチャート・分析コメントでは、以下の**6軸**にグルーピングした総合点（該当項目の平均）を用いる。IAF・6コアスキルに合わせた構成とする。

| 軸番号 | 軸名称（短いラベル） | 対応する項目ID | コアスキル対応 |
|--------|------------------------|----------------|----------------|
| 1 | 説明・設計 | E2, E3 | 説明力 |
| 2 | 見える化・編集 | E1, E4 | 情報編集力 |
| 3 | 場の観察 | B4 | 場の観察力 |
| 4 | 場のホールド・安心感 | B2, B3, B6, B1 | 場のホールド力等 |
| 5 | 問いかけ・リフレーミング | B5, B7, B8, B9, B10 | リフレーミング力・問いかけの定石 |
| 6 | 流れ・即興 | F1, F2, F3, F4 | 即興力・情報編集力（流れ） |

- **評価入力画面**では従来どおり E1〜F4 の18項目を表示する。6軸ごとにセクション見出しを付け、どの軸の質問か分かるようにグループ化して表示する。
- **結果画面・レーダー・分析コメント**では上記6軸の総合点を用いる。

## 5. 項目ID（データ・集計用）

- **鷹の目**: E1 ～ E4  
- **虫の目**: B1 ～ B10  
- **魚の目**: F1 ～ F4  

evaluations テーブルの scores（JSONB）では、上記 ID をキーにスコア値を格納する想定（例: `{"E1": 4, "E2": 5, ...}`）。**入力は18項目のまま不変**。6軸は集計・表示時のグルーピングのみに用いる。

## 6. 振り返り問い（6軸版）

結果画面で表示する振り返り用の固定問いは、6軸を前提に次のとおりとする。実装時の `reflectionQuestionsFixed.ts`（または同等の固定問い定義）は、この文案に合わせて差し替える対象とする。

| # | 問い | 意図 |
|---|------|------|
| 問い1 | 今回の場で、メインとして「うまくいった」と感じた瞬間はどこでしたか？ 参加者・サブの皆さんは、そのとき何を感じていましたか？ | うまくいった瞬間と見え方のすり合わせ。次の改善の土台を共有する。 |
| 問い2 | **6つの観点（説明・設計、見える化・編集、場の観察、場のホールド・安心感、問いかけ・リフレーミング、流れ・即興）のうち**、今日の振り返りで特に話したい視点はどれですか？ その視点で、メインに伝えたいことがあれば教えてください。 | 6軸から優先して深掘りするテーマを選び、具体的なフィードバックを引き出す。 |
| 問い3 | 次のファシリテーションで、メインが「1つだけ試してみる」としたら、何を試すと良さそうですか？ | 具体的な次のアクションを対話で決める。実行可能な一歩に落とす。 |
