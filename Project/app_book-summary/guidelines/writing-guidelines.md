# 要約執筆ガイドライン

## 📋 ガイドライン概要

このドキュメントでは、本の要約作成における執筆の基本原則と具体的な指針を定めています。
高品質で一貫性のある要約を作成するための基準として活用してください。

---

## 🎯 基本原則

### 1. 読者中心の視点
- **実務家向け**: デザイナー・ストラテジスト・組織開発者を想定
- **実践的価値**: 理論だけでなく実務での活用を重視
- **理解しやすさ**: 初心者にも分かりやすく、プロにも響く内容

### 2. 構造的な整理
- **論理的な構成**: 全体から詳細へ、理論から実践へ
- **視覚的な理解**: 表形式・矢印図を活用した整理
- **関連性の明確化**: 概念間の関係性を明確に示す

### 3. 品質の確保
- **正確性**: 著者の主張を正確に反映
- **完全性**: 重要な内容の漏れを防ぐ
- **一貫性**: 用語・表現の統一を保つ

---

## 📝 執筆スタイル

### 文章の基本スタイル

#### 1. 文体
- **敬語**: 「です・ます」調を基本とする
- **丁寧語**: 読者への配慮を表現
- **専門性**: 適切な専門用語の使用

#### 2. 文章の長さ
- **段落**: 1段落3〜5文程度
- **文**: 1文40〜60字程度
- **見出し**: 階層を明確にした構成

#### 3. 表現の統一
- **用語**: 同じ概念は同じ用語で統一
- **表記**: 数字・記号の表記を統一
- **敬語**: 文体の統一を保つ

### 具体的な表現指針

#### 1. 理論の背景（WHY）
```
この章では、[テーマ]について扱います。
著者は、[問題意識]という課題を認識し、
[テーマ]の重要性を指摘しています。
特に、[具体的な重要性]という点で、
実務家にとって重要な示唆を与えています。
```

#### 2. キー概念の説明
```
[概念名]とは、[簡潔な定義]のことです。
この概念は、[具体的な特徴]を持ち、
[実務での活用場面]で効果的に活用できます。
```

#### 3. 実務での使い方
```
[具体的な場面]では、以下の手順で活用できます：
1. [手順1]
2. [手順2]
3. [手順3]

特に重要なのは、[注意点・コツ]です。
```

#### 4. 拓也への問いかけ
```
あなたの[具体的な実務場面]では、
[本書の概念・手法]をどのように活用できますか？

また、[関連する課題]について、
どのような視点で取り組んでいますか？
```

## 📊 表形式の作成指針

### 1. 表の基本構成
- **列数**: 2〜4列程度（読みやすさを重視）
- **行数**: 3〜7行程度（重要概念に絞る）
- **内容**: 概念名と説明を明確に分離

### 2. 表の作成例
```markdown
| 概念 | 説明 | 実務での活用 |
|------|------|--------------|
| 概念A | 簡潔な定義 | 具体的な活用場面 |
| 概念B | 簡潔な定義 | 具体的な活用場面 |
| 概念C | 簡潔な定義 | 具体的な活用場面 |
```

### 3. 表作成の注意点
- **簡潔性**: 各セルの内容は簡潔に
- **一貫性**: 同じレベルの概念で構成
- **実用性**: 実務での活用を意識

---

## 🔄 矢印図の作成指針

### 1. 矢印図の基本構成
- **要素数**: 3〜5個程度（理解しやすさを重視）
- **関係性**: 因果関係・順序関係・包含関係
- **視覚性**: 直感的に理解できる構成

### 2. 矢印図の作成例
```markdown
```
[概念A] → [概念B] → [概念C]
    ↓         ↓         ↓
[説明A]   [説明B]   [説明C]
```

### 3. 矢印図作成の注意点
- **シンプル**: 複雑すぎない構成
- **論理的**: 関係性が明確
- **実用的**: 実務での応用がイメージできる

### デザイナー向け
- **UX視点**: ユーザー中心の考え方を強調

- **サービスデザイン**: サービス全体の設計視点

### 組織開発者向け
- **組織変革**: 組織の変化プロセス
- **チームビルディング**: チーム形成・発展

## 📋 品質チェック項目

### 内容の品質
- [ ] 著者の主張を正確に反映している
- [ ] 重要な内容の漏れがない
- [ ] 実務への応用が具体的
- [ ] 読者の理解を促進する構成

### 文章の品質
- [ ] 読みやすく、理解しやすい
- [ ] 専門用語の説明が適切
- [ ] 文体・表現が統一されている
- [ ] 論理的な構成になっている

### 実用性の品質
- [ ] 明日から使える具体的な方法
- [ ] 実務での活用場面が明確
- [ ] 注意点・コツが提示されている
- [ ] 段階的な実践ステップがある

## 🚨 避けるべき表現・構成

### 1. 避けるべき表現
- **抽象的な表現**: 「効果的」「重要」等の抽象語のみ
- **断定的な表現**: 「絶対」「必ず」等の過度に断定的な表現
- **専門用語の乱用**: 説明なしの専門用語の多用
- **冗長な表現**: 同じ内容の繰り返し

### 2. 避けるべき構成
- **論理の飛躍**: 説明なしの結論の提示
- **情報の過多**: 重要でない情報の過度な記載
- **構造の不明確**: 階層が分かりにくい構成
- **実用性の欠如**: 理論のみで実践方法がない

### 3. 改善すべき傾向
- **表面的な要約**: 深い洞察のない表面的な内容
- **独善的な解釈**: 著者の主張を歪める解釈
- **実務との乖離**: 実務で活用できない内容
- **読者視点の欠如**: 読者の立場を考慮しない内容
