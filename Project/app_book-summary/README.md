# 本の要約プロジェクト

## 📋 プロジェクト概要

このプロジェクトは、ビジネス書を「詳しく」「構造的に」「実践的に」要約するためのシステムです。
デザイン思考・組織開発・戦略構築に関心のある実務家向けに、高品質な要約を作成し、実践的な学びを提供します。

## 🎯 目的

- **高品質な要約**: 構造的で実践的な要約の作成
- **実務家向け**: デザイナー・ストラテジスト・組織開発者向けの内容
- **学習効果最大化**: 読書体験を超える価値の提供
- **知識の体系化**: 散在する情報の構造化と整理

## 📁 プロジェクト構成

```
Project/app_book-summary/
├── README.md                           # プロジェクト概要
├── templates/                          # テンプレート集
│   ├── summary-template.md             # 要約テンプレート
│   ├── chapter-template.md             # 章別要約テンプレート
│   └── highlight-template.md           # ハイライト整理テンプレート
├── guidelines/                         # ガイドライン
│   ├── summary-process.md              # 要約プロセス
│   ├── quality-standards.md            # 品質基準
│   └── writing-guidelines.md           # 執筆ガイドライン
├── examples/                           # 事例集
│   ├── success-cases.md                # 成功事例
│   └── faq.md                          # よくある質問
├── tools/                              # ツール・チェックリスト
│   ├── summary-checklist.md            # 要約チェックリスト
│   └── quality-checklist.md            # 品質チェックリスト
└── prompts/                            # プロンプト集
    ├── main-prompt.md                  # メインプロンプト
    ├── chapter-prompt.md               # 章別要約プロンプト
    └── highlight-prompt.md             # ハイライト整理プロンプト
```

## 🚀 使用方法

1. **要約開始時**: `templates/summary-template.md` を使用して要約を開始
2. **章別要約時**: `templates/chapter-template.md` で章ごとの詳細要約
3. **品質確認時**: `tools/` 内のチェックリストで品質を確認
4. **プロンプト使用時**: `prompts/` 内のプロンプトを活用

## 📊 主要指標

- 要約の構造性・実践性
- 読者の理解度・満足度
- 実務への応用可能性
- 知識の定着度
