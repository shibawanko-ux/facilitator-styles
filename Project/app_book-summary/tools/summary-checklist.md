# 要約作成チェックリスト

## 📋 チェックリスト概要

このチェックリストは、本の要約作成プロセスで確認すべき項目を段階別に整理したものです。
各段階で確実にチェックを行い、高品質な要約を完成させてください。

---

## 🔄 プロセス別チェックリスト

### 1️⃣ 準備段階チェック

#### 1.1 基本情報の確認
- [ ] **本の基本情報が正確**
  - タイトル、著者、出版社、出版年
  - 要約作成日、作成者名
- [ ] **目次情報の取得**
  - Amazonでの目次情報確認
  - 章構成の理解
- [ ] **ハイライト情報の整理**（該当する場合）
  - 色別重要度の確認
  - 章別の対応関係

#### 1.2 要約方針の決定
- [ ] **重点的に扱う章の特定**
  - 最も重要な章の選定
  - 実務家にとって価値のある章の特定
- [ ] **要約の深さ・詳細度の決定**
  - 全体要約の詳細度
  - 章別要約の詳細度
- [ ] **対象読者の明確化**
  - デザイナー、ストラテジスト、組織開発者等
  - 各読者層への配慮

#### 1.3 テンプレートの準備
- [ ] **テンプレートの選択**
  - `templates/summary-template.md` の使用
  - 本の情報をテンプレートに記入
- [ ] **プロンプトの準備**
  - `prompts/main-prompt.md` の確認
  - 必要に応じたカスタマイズ

---

### 2️⃣ 分析段階チェック

#### 2.1 全体構造の把握
- [ ] **本の全体構成の理解**
  - 著者の主張・問題意識の把握
  - 各章の関連性・流れの整理
- [ ] **主要概念の特定**
  - 重要な概念・用語の抽出
  - 概念間の関係性の整理
- [ ] **実務への応用可能性の検討**
  - ワークショップでの活用
  - プロジェクト運営での活用

#### 2.2 ハイライト情報の分析
- [ ] **色別重要度の確認**
  - ピンク：最重要（実務応用のヒント）
  - 青：重要（キー概念を裏づける視点）
  - 赤：補足（理論背景の補足）
- [ ] **ハイライトと章の対応関係**
  - 各章に関連するハイライトの整理
  - 要約への反映方針の決定

---

### 3️⃣ 執筆段階チェック

#### 3.1 全体要約の作成
- [ ] **3つのクリティカル・メッセージの抽出**
  - 本の核心を的確に表現
  - 実務家にとって価値のある内容
- [ ] **想定読者と得られることの明確化**
  - 対象読者の明確な定義
  - 読後に得られる視点・変化の具体化

#### 3.2 章別要約の作成
各章について以下を確認：

##### 3.2.1 理論の背景（WHY）
- [ ] **400〜500字で丁寧に説明**
  - 著者の問題意識の明確化
  - テーマの重要性の説明
  - 実務家にとっての価値の明確化

##### 3.2.2 キー概念と図式整理（WHAT）
- [ ] **表形式での概念整理**
  - 重要な概念の漏れがない
  - 説明が分かりやすく、実務的
  - 表の構成が論理的
- [ ] **矢印図での関係性整理**
  - 概念間の関係性が明確
  - 視覚的に理解しやすい
  - 実務への応用がイメージできる

##### 3.2.3 実務での使い方と問い（HOW）
- [ ] **具体的な活用方法の提示**
  - ワークショップでの活用例
  - 対話設計への応用
  - プロジェクト運営での活用
- [ ] **効果的な問いかけ例の提供**
  - 実践的な問いかけ
  - 深い気づきを促す問い

##### 3.2.4 拓也への問いかけ
- [ ] **自分ごととして考えられる問い**
  - 読者が自分の実務に当てはめて考えられる
  - 実務への応用を促す内容
  - 深い気づきを促す問い

##### 3.2.5 ハイライト一覧の整理
- [ ] **ハイライト情報の適切な活用**
  - ピンク・青のハイライトが重視されている
  - 要約内容との対応関係が明確
  - 補強・裏付け情報として機能している

---

### 4️⃣ 品質確認段階チェック

#### 4.1 内容チェック
- [ ] **3つのクリティカル・メッセージの明確性**
  - 本のエッセンスを的確に表現
  - 実務家にとって価値のある内容
- [ ] **各章の理論背景の適切性**
  - 著者の問題意識が明確
  - テーマの重要性が説明されている
- [ ] **キー概念の整理の分かりやすさ**
  - 重要な概念が漏れなく抽出
  - 説明が分かりやすく、実務的
- [ ] **実務での使い方の具体性**
  - 具体的な活用方法が提示
  - 明日から使える方法

#### 4.2 構造チェック
- [ ] **全体構成の論理性**
  - 全体から詳細への流れ
  - 理論から実践への流れ
- [ ] **章間の関連性**
  - 章の流れが自然
  - 関連性が明確
- [ ] **読みやすさ・理解しやすさ**
  - 段階的な説明
  - 専門用語の適切な説明

#### 4.3 実務性チェック
- [ ] **デザイナー向けの内容**
  - UX・サービス・組織デザインへの応用
  - デザイン思考の視点
- [ ] **ストラテジスト向けの内容**
  - 戦略構築への応用
  - 組織設計への活用
- [ ] **組織開発者向けの内容**
  - 組織開発・設計への応用
  - ワークショップ・ファシリテーションへの活用

---

### 5️⃣ 最終調整段階チェック

#### 5.1 全体のまとめ
- [ ] **最も重要な学びの抽出**
  - 本書全体を通して最も重要な学び
  - 実務家にとっての価値
- [ ] **実務への応用方針の提示**
  - 本書の学びを実務にどう活かすか
  - 具体的なアクションプラン
- [ ] **次のステップの提案**
  - 本書を読んだ後の次のアクション
  - 継続的な学習の方向性

#### 5.2 品質チェックリストの確認
- [ ] **品質チェックリストの全項目確認**
  - `tools/quality-checklist.md` の使用
  - 全項目の確認・修正
- [ ] **最終品質の確保**
  - 品質基準の達成確認
  - 必要に応じた修正

#### 5.3 ファイルの完成
- [ ] **メタデータの記入**
  - 作成日・更新日の記録
  - 作成者名・バージョン情報
- [ ] **参考資料の記載**
  - 使用したプロンプト
  - 参考資料・外部リンク

## 🚨 緊急対応項目

以下の項目に問題がある場合は、即座に修正してください：

### 重大な問題
- [ ] 著者の主張の重大な誤解
- [ ] 実務に有害な誤った情報
- [ ] 読者に誤解を招く表現
- [ ] 著作権侵害の可能性

### 中程度の問題
- [ ] 重要な内容の漏れ
- [ ] 実務での活用方法が不明確
- [ ] 構成の論理性が不十分
- [ ] 読みにくい文章

### 軽微な問題
- [ ] 表記の不統一
- [ ] フォーマットの不備
- [ ] メタデータの不備
- [ ] 参考資料の不備
