# 17 スキル分析テーブル 総合スコア行追加要件

スキル分析テーブル（表内）に**総合スコア**の行を追加し、実施前・直後・変化量(直後)（および Phase3 では 1ヶ月後・変化量(1ヶ月後)）を表示するためのプレースホルダーとデータ流れを規定する。

## 1. 対象マスタースライド

- **全体用（Phase 2）**: https://docs.google.com/presentation/d/1lBDBCjMu9iiRcdTSgUwNycDbTPAXfZLT6cPtT87EWcw/edit
- **個人用**: https://docs.google.com/presentation/d/1hNVB6KZe9-VeYzSO_FY2RggGkM6XPiKXkqgzEdTLblI/edit

上記スライドの「スキル分析テーブル（表内）」に、総合スコア用の行を追加する。

## 2. 追加するプレースホルダー

### 2.1 スライド1: エグゼクティブサマリー（該当なし）／スライド2: スキル分析（レーダーチャート）

スライド2の**スキル分析テーブル（表内）**に、総合スコアの行を追加する。

| プレースホルダー | データ内容 | データソース |
|-----------------|------------|------------------------|
| `{{CgrF_1}}` | 総合スコア 実施前 | 01_エグゼクティブサマリー.csv（総合スコア行 > 実施前） |
| `{{CgrF_2}}` | 総合スコア 直後 | 01_エグゼクティブサマリー.csv（総合スコア行 > 直後） |
| `{{CgrF_3}}` | 総合スコア 1ヶ月後（Phase3） | 01_エグゼクティブサマリー.csv（総合スコア行 > 1ヶ月後）。Phase2 では `-` |
| `{{CgrF_4}}` | 総合スコア 変化量(直後) | 01_エグゼクティブサマリー.csv（総合スコア行 > 変化量(直後)） |
| `{{CgrF_5}}` | 総合スコア 変化量(1ヶ月後)（Phase3） | 01_エグゼクティブサマリー.csv（総合スコア行 > 変化量(1ヶ月後)）。Phase2 では `-` |

- **命名規則**: 既存の CgrA〜CgrE（5軸スキル）に続く **F = 総合スコア**。`_1`〜`_5` の意味は 13 仕様と同様（実施前・直後・1ヶ月後・変化量(直後)・変化量(1ヶ月後)）。
- **表の表示**: 「総合スコア」行として、実施前・直後・変化量(直後)（および Phase3 では 1ヶ月後・変化量(1ヶ月後)）を表示する。

### 2.2 スライド3: 組織別分析

各組織の**スキル分析テーブル（表内）**に、総合スコアの行を追加する。

| プレースホルダー | データ内容 | データソース |
|-----------------|------------|------------------------|
| `{{OgrF_1}}` | 該当組織の総合スコア 実施前 | pre_data を所属でフィルタし、総合スコア（total）を算出 |
| `{{OgrF_2}}` | 該当組織の総合スコア 直後 | 02_組織別分析.csv（総合スコア列）または post_data から算出 |
| `{{OgrF_3}}` | 1ヶ月後（Phase3） | Phase2 では `-` |
| `{{OgrF_4}}` | 該当組織の総合スコア 変化量(直後) | 直後 − 実施前 |
| `{{OgrF_5}}` | 変化量(1ヶ月後)（Phase3） | Phase2 では `-` |

- **命名規則**: 既存の OgrA〜OgrE に続く **OgrF = 組織別総合スコア**。
- **データ取得**: 組織別の実施前 total は `analyze_by_department(pre_data)` の該当部署の `total`。直後は `analyze_by_department(post_data)` の該当部署の `total` または 02_組織別分析.csv の総合スコア列。

### 2.3 個人ごとのデータ（スライド挿入内容（個別））

個人別スライドの**スキル分析テーブル（表内）**に、総合スコアの行を追加する。

| プレースホルダー | データ内容 | データソース |
|-----------------|------------|------------------------|
| `{{CgrF_1_P}}` | 当該参加者の総合スコア 実施前 | 生成レポート（個別）.md（スコア推移表 > 総合スコア > 実施前） |
| `{{CgrF_2_P}}` | 当該参加者の総合スコア 直後 | 生成レポート（個別）.md（スコア推移表 > 総合スコア > 直後） |
| `{{CgrF_3_P}}` | 1ヶ月後（Phase3） | Phase2 では未使用または `-` |
| `{{CgrF_4_P}}` | 当該参加者の総合スコア 変化量(直後) | 直後 − 実施前 |
| `{{CgrF_5_P}}` | 変化量(1ヶ月後)（Phase3） | Phase2 では未使用または `-` |

- **命名規則**: 個人用サフィックス `_P`。既存の CgrA_1_P〜CgrE_5_P に続く **CgrF_*_P = 個人の総合スコア**。
- **データ取得**: 個人別スコア推移データの `pre_total`（実施前総合）、`post_total`（直後総合）、`follow_total`（1ヶ月後、Phase3）、および変化量を算出。

## 3. データフロー

- **01_エグゼクティブサマリー.csv**: 既に「総合スコア」行が出力されている（`generate_executive_summary_csv`）。この行をパースし、CgrF_1〜CgrF_5 に対応する値をスライド挿入内容MD・GAS に渡す。
- **組織別**: `analyze_by_department` の戻り値に各部署の `total` が含まれる。実施前は `dept_analysis_pre`、直後は 02_組織別分析.csv または `dept_analysis_post` から取得。OgrF_1〜OgrF_5 を各組織ブロックのスキル分析テーブルに追加する。
- **個人別**: 個人別スコア推移の `pre_total`・`post_total`・`follow_total` および変化量から CgrF_1_P〜CgrF_5_P を出力する。

## 4. 実装対象と整合性

| 対象 | 変更内容 |
|------|----------|
| スライド挿入内容（全体）MD | スライド2セクションに CgrF_1〜CgrF_5 の表行を追加。スライド3各組織に OgrF_1〜OgrF_5 の表行を追加。 |
| スライド挿入内容（個別）MD | 各参加者セクションのスキル分析テーブルに CgrF_1_P〜CgrF_5_P の表行を追加。Phase2 では CgrF_3_P・CgrF_5_P は `-`。 |
| report_generator.py | 01 CSV パースで「総合スコア」行を読み skill_scores['total'] を設定。CgrF_* の表行を出力。組織別 ogr_rows に F（総合スコア）を追加。個人用で CgrF_*_P を出力。 |
| gas_generator.py | PLACEHOLDER_MAP に CgrF_1〜CgrF_5 を追加。parse_slide_content_markdown の skill_scores に CgrF_* を反映（既存の Cgr パースで `CgrF` も拾うよう対応）。 |
| gas_generator.py（組織別） | PLACEHOLDER_MAP に OgrF_1〜OgrF_5 を追加。buildOrgSlidesFromCSV_ の orgDict に getOgr('F', 1)〜getOgr('F', 5) を追加。 |
| individual_gas_generator.py | 参加者 placeholders に CgrF_1_P〜CgrF_5_P を追加。 |
| parse_organization_data_from_markdown | 行検出条件は既に `'{{O' in line` のため、OgrF_* も表行としてパースされ ORGANIZATION_DATA に含まれる。追加変更不要（キーが MD に存在すればよい）。 |

## 5. Phase 別

- **Phase 2**: CgrF_3 / CgrF_5、OgrF_3 / OgrF_5、CgrF_3_P / CgrF_5_P は `-` または未使用。
- **Phase 3**: 1ヶ月後・変化量(1ヶ月後) もデータがあれば算出・表示する。

## 6. 実装可否の結論

総合スコア行の追加は**可能**です。必要なデータ（01 CSV の総合スコア行、組織別の total、個人の pre_total/post_total）は既に存在しており、既存の Cgr/Ogr の仕組みを「F＝総合スコア」で拡張する形で実装できます。実装時は本要件に沿って進めてください。
