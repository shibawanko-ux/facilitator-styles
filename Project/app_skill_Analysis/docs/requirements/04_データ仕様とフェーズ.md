# 04 データ仕様とフェーズ

## 1. 用語統一表

アプリ・ドキュメント全体で以下を統一する。

### 1.1 測定フェーズ（実施プロセス）

| 略称 | 日本語 | 説明 |
|------|--------|------|
| PreWS | 実施前 | 本人の事前自己評価 |
| PostWS | 直後 | 本人の直後自己評価＋満足度等 |
| Follow-up | 1ヶ月後 | 本人の1ヶ月後評価＋上長評価 |

### 1.2 アプリのPhase（分析レポートの種類）

| Phase | 必要なCSV | 生成するレポート | 備考 |
|-------|-----------|------------------|------|
| Phase 1 | 実施前のみ | 現状診断レポート（任意） | **GAS 出力は行わない**。Phase 1 専用のアウトプット（GAS およびそれのみに関する出力）は**必須としない**（しなくてもよい）。 |
| Phase 2 | 実施前＋直後 | 学習効果レポート | GAS 連携あり |
| Phase 3 | 実施前＋直後＋1ヶ月後（上長1ヶ月後は任意） | 最終定着度レポート（ギャップ分析・推奨プログラム含む） | GAS 連携あり |

### 1.3 評価軸とキー・質問の対応

| 軸（日本語） | 英語キー | 質問ID |
|-------------|----------|--------|
| リサーチ・分析力 | research | Q1, Q2, Q3 |
| 構想・コンセプト力 | concept | Q4, Q5, Q6 |
| 具体化・検証力 | delivery | Q7, Q8, Q9 |
| 伝達・構造化力 | communication | Q10, Q11, Q12 |
| 実現・ディレクション力 | implementation | Q13, Q14, Q15 |

## 2. CSVファイル形式

### 2.1 実施前.csv / 直後.csv / 1ヶ月後.csv

**必須カラム**:
- `メールアドレス`: 参加者のメールアドレス（個人識別用）
- `氏名`: 参加者の氏名
- `所属部署`: 所属部署名
- `Q1` 〜 `Q15`: 各質問のスコア（1〜5の数値）

**オプション（直後.csv）**:
- `WS満足度`, `WS理解度`, `NPS`, `Q16A`（活用意欲）等。フォームの質問文が列名になっている場合（例: 「今回のワークショップの内容に対する満足度はどの程度ですか？」）は、07_フォーム準拠_CSV取り込み仕様に従い正規化して上記カラム名として扱う。

**オプション（1ヶ月後.csv）**:
- `Q16B`: 実践頻度（1: よくあった、2: たまにあった、3: ほとんどなかった、4: 全くなかった）

### 2.2 上長1ヶ月後.csv

**必須カラム**:
- `対象者メールアドレス`: 評価対象者のメールアドレス
- `M1` 〜 `M5`: 各スキル軸の上長評価（1〜5の数値）。M1=research, M2=concept, M3=delivery, M4=communication, M5=implementation

## 3. Phase の自動判定条件

アップロードされたファイルの有無に基づき、以下で判定する。

| 条件 | 判定結果 |
|------|----------|
| 実施前のみ | Phase 1 |
| 実施前＋直後 | Phase 2 |
| 実施前＋直後＋1ヶ月後 | Phase 3（上長1ヶ月後は任意。なくても Phase 3） |

- 実施前データは**必須**。実施前がなければエラーとする。
- エンコーディングは UTF-8 を想定する（BOM 付きの場合は読込時に除去する）。
- フォームからエクスポートした CSV（質問文が列名・日本語が回答値）もアップロード可能。その場合は **07_フォーム準拠_CSV取り込み仕様** に従い、読込直後に列名・値を正規化してから上記の形式として扱う。
