# 28 Phase3 テキスト生成 フェーズ別差別化要件

スライド生成において、**Phase2（研修直後）の文脈がPhase3（1ヶ月後）レポートに混入する**問題を防ぐための要件を規定する。
Phase3では「1ヶ月後に実際どうだったか」を中心に据え、Phase2特有の要素（活用意欲・アクション宣言）を除外する。

---

## 1. 背景・課題

### 1.1 Phase2とPhase3の文脈の違い

| 要素 | Phase2（直後） | Phase3（1ヶ月後） |
|------|--------------|----------------|
| 主軸 | 「これからやりたい」意向・意欲 | 「実際にやった」実績・定着 |
| 活用意欲 | ✅ 掲載する（`post_data`） | ❌ 掲載しない（意向は既に過去） |
| アクション宣言 | ✅ 掲載する（`post_data`） | ❌ 掲載しない（宣言も過去の話） |
| スコア比較 | 実施前→直後 | 実施前→直後→1ヶ月後（3時点） |
| 実践エビデンス | 不要 | ✅ Q17B（`follow_data`）を引用 |
| 上長コメント | 不要 | ✅ M7（`manager_data`）を引用 |
| 評価視点 | 研修効果の確認 | 現場定着の確認 + 今後の示唆 |

### 1.2 混入が発生しやすい箇所

- `generate_radar_analysis_summary_impl`：`phase >= 2` の条件で活用意欲・アクション宣言ブロックが Phase3 でも発火していた
- `generate_executive_strength_text_impl`：`post_data` のアクション宣言を Phase3 でも引用していた
- `generate_executive_weakness_text_impl`：「補強の機運がうかがえる」等、Phase2の意向ベース表現が残っていた
- `extend_strength_text_impl` / `extend_weakness_text_impl`：活用意欲・アクション宣言が Phase3 でも出力されていた

---

## 2. 基本方針

### 2.1 Phase3テキストの構成原則

```
① 3時点変容（事実）: 実施前→直後→1ヶ月後のスコア推移を数値で示す
② 解釈（なるほど感）: 推移が何を意味するかを1文で述べる
③ 定着スキル（ポジティブ）: 1ヶ月後に最もスコアが高い/向上した軸を具体的に
④ 伸びしろ（課題）: 1ヶ月後に最もスコアが低い軸と今後の示唆
⑤ 実績エビデンス: Q17B（実践報告）または M7（上長コメント）を1件引用
```

### 2.2 Phase3で出力してはいけない内容

- 「〇名中〇名が活用意欲『湧いている』以上と回答」（post_data の意欲指標）
- 「〇〇といったアクション宣言が見られ」（post_data のアクション宣言引用）
- 「補強の機運がうかがえる」（意向ベースの表現）
- 「学習内容の定着が期待される」（Phase2の締め文として使われがちな表現）

### 2.3 Q17B実践エビデンスの採用条件

ネガティブ・ポジティブ両方のキーワードチェックを実施し、**両条件を満たすものだけ**採用する。

**除外キーワード（いずれか1つでも含む場合は不採用）**：
`なかった` `無かった` `機会がない` `機会がなかった` `機会が無かった`
`機会はなかった` `機会はない` `実践できていない` `活用できていない`
`難しかった` `難しい` `忙しく` `忙しい` `できなかった` `課題` `問題`
`障壁` `まだ` `少ない` `不足`

**採用キーワード（いずれか1つ以上含む場合のみ採用）**：
`実践` `できた` `活用` `向上` `改善` `取り組んだ` `取り組み` `実施`
`意識` `取り入れ` `スムーズ` `深く` `考えられる` `伝えられる`
`使えた` `使った` `活かせた` `活かした` `やってみた` `試みた`

**最小文字数**：15文字以上

実装：`_extract_q17b_evidence()` 関数（`report_generator_impl.py`）

## 3. プレースホルダー別 Phase3 要件

### 3.1 `{{S_block_1_body}}`（エグゼクティブサマリー 総評）

**データソース**：`generate_summary_comment_impl`（Phase3ブロック）

| # | 構成要素 | 内容 |
|---|---------|------|
| Bullet 1 | 3時点変容サマリー | 「実施前→直後→1ヶ月後の推移 + 研修効果の定着確認」 |
| Bullet 2 | 定着スキル + 伸びしろ | 「1ヶ月後に最も定着した軸（実施前比）+ 伸びしろとして残る軸」 |
| Bullet 3 | （任意）Q17B実践エビデンス | ポジティブ条件を満たす Q17B コメントがあれば引用 |

**除外**：活用意欲、アクション宣言（`post_data`）、満足度・理解度（Phase3では不要）

### 3.4 `{{C_block_1_body}}`（スキル分析 分析総評）

**データソース**：`generate_radar_analysis_summary_impl`（Phase3専用ブロック）

| # | 構成要素 | 内容 |
|---|---------|------|
| 1 | 3時点変容 | 「実施前X.XX点→直後X.XX点→1ヶ月後X.XX点と推移し、研修効果が現場実践に定着（実施前比±XXpt）」 |

| 4 | 今後の示唆 | 「『〇〇（伸びしろ軸）』を重点課題として、上長との対話と業務での意識的な実践を継続することが期待される」 |

**除外**：活用意欲、アクション宣言、満足度・理解度

## 4. スコア変容テキストのパターン定義

### 4.1 強み（O_block_2_body）

| 条件 | テキスト |
|------|---------|
| `follow > post + 0.05` | 「直後のX.XX点から1ヶ月後にX.XX点へ向上しており、実践の定着が確認できる。」 |
| `follow < post - 0.05` | 「直後のX.XX点から1ヶ月後にX.XX点と変動が見られる。継続的な実践で定着が期待される。」 |
| その他（横ばい） | 「直後のX.XX点から1ヶ月後もX.XX点と高いスコアを維持しており、研修効果の持続が確認できる。」 |

### 4.2 伸びしろ（O_block_3_body）

| 条件 | テキスト |
|------|---------|
| `follow > post + 0.05` | 「直後のX.XX点から1ヶ月後にX.XX点へ改善しており、引き続き意識的な実践を継続することで更なる向上が期待できる。」 |
| `follow < post - 0.05` | 「直後のX.XX点から1ヶ月後にX.XX点と伸びが見られていない。意識的な実践機会を増やすことが重要な課題である。」 |
| その他（横ばい） | 「直後のX.XX点から1ヶ月後もX.XX点と横ばいで推移しており、さらなる実践の積み重ねが求められる。」 |

## 8. 今後の注意事項

- Phase2 → Phase3 の移植時に **`phase >= 2` の条件を `phase == 2` に変更し忘れる**ことが混入の典型パターン。新しいテキスト生成ロジックを追加する際は必ず Phase 分岐を意識する。
- Q17B の採用条件（除外・採用キーワードリスト）は実データを見ながら随時更新する。特に「〇〇の機会が**無かった**」のような漢字バリアントは見落としやすいため注意。
- 上長コメント（M7）は `対象者メールアドレス` → `follow_data` のメール → 部署名で照合する。`manager_data` に部署列がない場合はこの間接参照が必要。
