# 12 C_block コメント要件

スライド2「全社スキル分析（レーダーチャート）」の `{{C_block_1_body}}`・`{{C_block_2_1_body}}`・`{{C_block_2_2_body}}` について、文章量の目標と読み手の納得度を高めるための要件を規定する。

---

## 1. 概要・目的

### 1.1 対象

スライド2（LAYOUT ID: radar_chart）は、全社の5軸スキルをレーダーチャートで可視化し、実施前・直後（・1ヶ月後）の推移を示す。以下の3つのプレースホルダーに分析コメントが挿入される。

| プレースホルダー | 用途 | データソース |
|----------------|----------|-------------|
| `{{C_block_1_body}}` | 分析総評 | 生成レポート.md（2. 全社スキル分析 > 分析総評） |
| `{{C_block_2_1_body}}` | レーダーチャート説明 > 実施前（青線） | 生成レポート.md（2. 全社スキル分析 > レーダーチャートの説明 > 実施前） |
| `{{C_block_2_2_body}}` | レーダーチャート説明 > 直後（オレンジ線） | 生成レポート.md（2. 全社スキル分析 > レーダーチャートの説明 > 直後） |

### 1.2 目的

- **文章量の拡充**: 現状の簡潔な説明を、読み手が分析に対して納得感を持てる程度に拡充する。
- **納得度の向上**: 根拠の明示、解釈の説明、示唆を含めることで、「なぜその分析か」が伝わるコメントとする。

---

## 2. 対象と文章量一覧

| プレースホルダー | 目標文章量 | 現在の目安 |
|----------------|-----------|-----------|
| `{{C_block_1_body}}` | **約400文字** | 約150〜200文字 |
| `{{C_block_2_1_body}}` | **約300文字** | 約40〜60文字 |
| `{{C_block_2_2_body}}` | **約300文字** | 約100〜130文字 |

※ 文字数は改行・スペースを含む。目標は目安であり、±50文字程度の許容とする。

---

## 3. 納得度を高めるための指針

10_分析コメント品質要件と同様、以下の要素を含める。

- **根拠の明示**: スコア、変化量、割合などを具体的に示す。
- **解釈の説明**: その数値や傾向が何を意味するかを簡潔に述べる。
- **示唆・次のアクション**: 強みをどう活かすか、弱みをどう補うかの方向性（簡潔に）。
- **論理の流れ**: 現状 → 根拠 → 解釈 → 示唆 の順で構成することを推奨。

---

## 4. プレースホルダー別詳細要件

### 4.1 `{{C_block_1_body}}`（分析総評）

- **文章量**: 約400文字
- **内容要素**:
  1. 組織のスキル特性（バランス型・実行部隊型・強み突出型等）の変化
  2. 実施前→直後（→1ヶ月後）のスコア変化の要約と、向上幅が大きいスキル軸の具体的な数値
  3. 学習効果の確認と、今後の期待・示唆
  4. （Phase 2以上）活用意欲の集計（湧いている以上: ○名/○名）と代表的なアクション宣言1〜2件の引用
  5. （Phase 2以上）満足度・理解度を補足し、スコア向上が学習効果によることを補強
- **出力元**: `generate_radar_analysis_summary_impl`（`report_generator_impl.py`）
- **形式**: 段落形式。スライド表示に適した改行・句読点を使用。

### 4.2 `{{C_block_2_1_body}}`（実施前の説明）

- **文章量**: 約300文字
- **内容要素**:
  1. 実施前の5軸スコアの概要（総合スコア、各軸の数値）
  2. 最低スキル・最高スキルの特定と、その意味（課題・強み）
  3. 実施前の組織タイプ・傾向（例: 実行部隊型、リサーチが弱い等）
  4. バランスの評価（偏りの有無、改善の余地）
- **出力元**: `generate_radar_description_pre_extended_impl`（`report_generator_impl.py`）
- **形式**: 段落形式。

### 4.3 `{{C_block_2_2_body}}`（直後の説明）

- **文章量**: 約300文字
- **内容要素**:
  1. 直後の5軸スコアの概要（総合スコア、各軸の数値）
  2. 実施前との比較・変化量（特に向上幅が大きい軸を具体的に）
  3. バランスの改善状況（偏りの解消、均整化）
  4. 学習効果の示唆（ワークショップによる定着の期待）
- **出力元**: `generate_radar_description_post_extended_impl`（`report_generator_impl.py`）
- **形式**: 段落形式。Phase 1 の場合は空または簡易版とする。

## 5. 実装対象とデータフロー

| ファイル | 変更内容 |
|----------|----------|
| `src/report_generator_impl.py` | `generate_radar_analysis_summary_impl` の拡張（post_data, satisfaction を受け取り約400文字）。`generate_radar_description_pre_extended_impl`, `generate_radar_description_post_extended_impl` を新規追加（各約300文字） |
| `src/report_generator.py` | `generate_radar_analysis_summary` に post_data, satisfaction を渡す。`generate_report_markdown` および `generate_slide_content_markdown` で拡張版のレーダー説明関数を使用 |
| `スライド挿入内容_*.md` | 再生成により反映 |
| `GASコード_*.gs` | 再生成により反映 |

### 5.1 データソース

- **analysis**: pre, post, follow の各スコア（research, concept, delivery, communication, implementation, total）
- **post_data**: 直後.csv（Phase 2以上）。Q16A, Q17A を活用
- **satisfaction**: 満足度・理解度の平均（Phase 2以上）。analysis.get('satisfaction') から取得

---

## 6. Phase 別の適用

- **Phase 1**: 直後データがないため、C_block_1_body は実施前のみの分析。C_block_2_2_body は空または「-」。
- **Phase 2**: 上記すべての要件を適用。post_data, satisfaction を活用。
- **Phase 3**: Phase 2 と同様。加えて1ヶ月後データを分析総評に反映可能（将来拡張）。
