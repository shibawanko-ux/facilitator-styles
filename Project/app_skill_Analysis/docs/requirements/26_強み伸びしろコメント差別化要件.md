# 26 強み・伸びしろコメント差別化要件

スライド1「エグゼクティブサマリー」の **`{{S_block_2_body}}`（強み）** と **`{{S_block_3_body}}`（伸びしろ）** について、**強みと伸びしろが同様に見えない**ようにし、読み手が「**強いである**」「**伸びしろである**」を一読して理解できるようにする要件を規定する。部長など役職者の納得感を高める。

---

## 1. 目的・背景

### 1.1 やりたいこと

- **強み**と**伸びしろ**を、読み手が「どちらが強みか／伸びしろか」迷わず理解できるようにする。
- 両ブロックの**言い回し・構成・トーン**を差別化し、「強みと伸びしろが似た文章」に見えないようにする。
- 特に**部長など役職者**が一読して「強みはここ」「伸びしろはここ」と納得し、次のアクションを考えやすい表現にする。

### 1.2 現状の課題

- **同一のアクション宣言が両方に登場する**  
  例：「答えを誘導するような質問をしない」が強みブロックと伸びしろブロックの両方に出ており、どちらの文脈か分かりにくい。
- **締めの型が似ている**  
  「満足度・理解度も高く、〜が期待される／意欲は高い」など、強み・伸びしろで同じパターンになりがち。
- **役割の違いが伝わりにくい**  
  「強み＝現状の強さ・活かし方」「伸びしろ＝補うと効果が大きい領域・次の一手」の対比が弱い。

### 1.3 対象

| 対象 | 説明 |
|------|------|
| スライド | スライド1：エグゼクティブサマリー |
| プレースホルダー | `{{S_block_2_body}}`（強み）、`{{S_block_3_body}}`（伸びしろ） |
| 読み手 | 部長・マネジメントなど、傾向と次の一手を重視する役職者 |

---

## 2. 方針（差別化と納得感）

### 2.1 強みと伸びしろの「役割」を明確に分ける

- **強み（S_block_2）**  
  「いま組織として持っている強さ」と「それをどう活かすか」を伝える。読んだ人が「ここが強みだ」「この強みを活かそう」と理解できるようにする。
- **伸びしろ（S_block_3）**  
  「補うと効果が大きい領域」と「だから何をするとよいか」を伝える。読んだ人が「ここを伸ばすとよい」「この方向で手を打とう」と理解できるようにする。

### 2.2 表現上の差別化

- **強み**  
  肯定的・評価のトーン。「〜が強みとして表れている」「〜が高い」など。強み**そのもの**と、強みの**業務での効き方・活かし方**を前面に出す。アクション宣言を使う場合は「強みを活かす・定着させる」文脈に限定する。
- **伸びしろ**  
  課題・可能性のトーン。「〜に伸びしろがある」「〜に余地がある」「補強が期待される」など。伸びしろ**そのもの**と、**補うと得られる効果・次の一手**を前面に出す。アクション宣言を使う場合は「伸びしろを補う・改善する」文脈に限定し、強みでは使わない表現にする。

### 2.3 部長の納得感

- **一文目で役割が分かる**  
  強み：冒頭で「何が強みか」が一言で分かる（例：「〜が組織の強みとして表れている」）。伸びしろ：冒頭で「何が伸びしろか」が一言で分かる（例：「〜に伸びしろがある」「〜の補強が期待される」）。
- **根拠は簡潔に**  
  スコア・満足度・参加者数などは「補足」に留め、長い数値羅列は避ける。
- **次の一手が分かる**  
  強みは「維持・活用」、伸びしろは「補強・改善」の方向が、それぞれ1文程度で伝わるようにする。

## 3. プレースホルダー別要件

### 3.1 `{{S_block_2_body}}`（強み）

- **目的**  
  読み手が「**ここが強みだ**」と一読して理解し、部長が「この強みを活かそう」と納得できる内容にする。
- **含めるべき内容（構成の目安）**
  1. **強みの本質を一文で**  
     「〜が組織の強みとして表れている」「〜力が強みである」など、**軸名・点数より「何が強みか」**を先に書く。
  2. **強みが業務・組織にどう効いているか**  
     「その強みが、現場・チーム・成果にどうつながっているか」を1文で示す。
  3. **強みの維持・活用の方向（任意）**  
     強みを定着・活用するための示唆を1文程度。参加者のアクション宣言を使う場合は、**強みと関連する宣言のみ**とし、「強みの現場活用が期待される」など強み文脈で締める。
- **避けること**
  - 伸びしろブロックと同じアクション宣言をそのまま流用すること。
  - 「満足度・理解度も高く、**改善**意欲は高い」など、伸びしろ寄りの表現を強みブロックで使うこと。
- **差別化の目安**
  - 強み専用の締め（例：「強みの定着が期待される」「強みを活かした実践が期待される」）。伸びしろブロックでは使わない「強み」キーワードを入れる。
- **文章量**  
  約250〜350文字（スライド表示に収まる長さ）。

### 3.2 `{{S_block_3_body}}`（伸びしろ）

- **目的**  
  読み手が「**ここが伸びしろだ**」と一読して理解し、部長が「ここを伸ばすと効果的だ」と納得できる内容にする。
- **含めるべき内容（構成の目安）**
  1. **伸びしろの本質を一文で**  
     「〜に伸びしろがある」「〜の補強が期待される」など、**軸名・点数より「何が伸びしろか」**を先に書く。
  2. **伸びしろを補うと何が良くなるか**  
     「その領域を強化すると、業務・組織にどんな効果があるか」を1文で示す。
  3. **改善の方向性・次の一手（任意）**  
     振り返り・役割分担・次の学習など、補強の方向を1文程度。参加者のアクション宣言を使う場合は、**伸びしろ（弱み軸）と関連する宣言のみ**とし、「補強の機運がうかがえる」「改善を志向した宣言があり」など伸びしろ文脈で締める。
- **避けること**
  - 強みブロックと同じアクション宣言をそのまま流用すること。
  - 「強みの定着が期待される」など、強みブロックと同一の締め型を使うこと。
- **差別化の目安**
  - 伸びしろ専用の締め（例：「補強の機運がうかがえる」「伸びしろを補強する意欲がうかがえる」）。強みブロックでは使わない「伸びしろ」キーワードを入れる。
- **文章量**  
  約250〜350文字（スライド表示に収まる長さ）。

## 4. 強み・伸びしろの「同様に見えない」ためのルール

- **同一アクション宣言の重複禁止**  
  ある参加者のアクション宣言を引用する場合、**強みブロックか伸びしろブロックのどちらか一方のみ**で使用する。強み軸に関連する宣言は S_block_2 のみ、伸びしろ軸（弱み軸）に関連する宣言は S_block_3 のみに出す。
- **締めの型の分離**  
  強みは「強み・定着・活用・活かす」などの語で締める。伸びしろは「補強・改善・伸びしろ・意欲」などの語で締める。同じ「満足度・理解度も高く、〜」でも、強みは「強みの定着」、伸びしろは「改善意欲・補強の機運」など、**最後の着地点を分ける**。
- **冒頭の型を分ける**  
  強みは必ず「強み」「強みとして」などを含む一文で始める。伸びしろは必ず「伸びしろ」「補強」「余地」などを含む一文で始める。

## 5. 受入基準（例）

- **強み（S_block_2）**  
  - 一読して「ここが強みだ」と分かる。強みの本質・効き方・維持・活用の方向が書かれている。伸びしろブロックと同じアクション宣言がそのまま重複していない。締めが「強み」文脈（定着・活用など）になっている。
- **伸びしろ（S_block_3）**  
  - 一読して「ここが伸びしろだ」と分かる。伸びしろの本質・補う効果・改善の方向が書かれている。強みブロックと同じアクション宣言がそのまま重複していない。締めが「伸びしろ」文脈（補強・改善意欲など）になっている。
- **部長の納得感**  
  - 強みと伸びしろが、構成・トーン・キーワードで明確に区別されている。それぞれ「強いである」「伸びしろである」ことが、読み手に伝わる。
