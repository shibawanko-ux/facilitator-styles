# 24 個別レーダーチャート説明の傾向分析要件

スライド挿入内容（個別）の「レーダーチャートの説明」において、**数字を列挙するのではなく傾向を文章で描写する**ための要件を規定する。

対象プレースホルダーは以下の3つで、それぞれ生成ロジックが異なる。

---

## 1. 背景・課題

### 1.1 対象プレースホルダー

| プレースホルダー | 用途 | データソース |
|----------------|----------|-------------|
| `{{C_block_2_1_body_P}}` | レーダーチャート説明 > 実施前（青線） | 生成レポート（個別）.md（〇〇様 > レーダーチャートの説明 > 実施前） |
| `{{C_block_2_2_body_P}}` | レーダーチャート説明 > 直後（オレンジ線） | 生成レポート（個別）.md（〇〇様 > レーダーチャートの説明 > 直後） |
| `{{C_block_2_3_body_P}}` | レーダーチャート説明 > 1ヶ月後（緑線）※Phase3のみ | 生成レポート（個別）.md（〇〇様 > レーダーチャートの説明 > 1ヶ月後） |

> ⚠️ **注意（命名規則）**: `{{C_block_2_3_body_P}}` は `_body` を含む正式名称。過去に `{{C_block_2_3_P}}`（`_body` 抜け）と誤記されていたバグあり（2026-03-11修正済み）。`report_generator.py` でのスライド挿入内容（個別）Markdown生成時も必ず `_body` を含める。

### 1.2 現状の課題

- **C_block_2_2_body_P（直後の説明）** が、参加者ごとに**数字は違うが文章構造がすべて同じ**になっている。
  - 現状の出力例: 「全体的に向上し、バランスが改善（リサーチ・分析力X.XX点、構想・コンセプト力X.XX点、…）」
  - 問題: **実施前との比較**がなく、「どの軸がどれだけ伸びた／横ばい／低下したか」といった**傾向分析**が含まれていない。そのため、数字は違っても「同じ説明の型」に聞こえ、参加者ごとの変化の特徴が伝わりにくい。

### 1.3 やりたいこと

- **実施前と直後の数字を比較しての傾向分析**を、`{{C_block_2_2_body_P}}`（および必要に応じて `{{C_block_2_1_body_P}}`）に反映する。
- 参加者ごとの**スコア変化の特徴**（大きく伸びた軸、横ばいの軸、やや低下した軸など）が分かる簡潔な説明文とする。

---

## 2. 要件概要

### 2.1 方針

- **C_block_2_2_body_P（直後の説明）**: 実施前スコアと直後スコアを比較し、**傾向分析**を主とした文章にする。
  - 直後の各軸の点数を列挙するだけではなく、「実施前→直後でどう変わったか」を述べる。
  - 例: 総合・各軸の変化量（+0.5pt、横ばい、-0.2pt 等）や、**特に伸びた軸／変化が小さかった軸／低下した軸**を簡潔に記載する。
- **C_block_2_1_body_P（実施前の説明）**: 現状でも「最も低い／高い軸」等で一定の差別化はある。必要に応じて、直後との対比を意識した表現に拡張可能（本要件では直後説明の傾向分析を優先）。

### 2.2 文章量・形式

- スライド1行〜数行で収まる程度（長文にしない）。表セルや GAS での置換を考慮し、**改行は入れず1文〜2文程度**を目安とする（`_cell_safe` により改行は除去される前提）。
- 文字数目安: **おおむね 80〜150 文字**（読みやすさを保ちつつ、傾向が伝わる長さ）。

---

## 3. 内容要件（C_block_2_2_body_P）

### 3.1 必須で含める要素

1. **実施前と直後の比較**
   - 総合スコアの変化（実施前 X.XX 点 → 直後 Y.YY 点、+Z.ZZpt 等）。
2. **傾向の要約**
   - **特に伸びた軸**: 変化量が大きいスキル軸を1〜2軸、具体的な変化量（例: +0.67pt）とともに記載。
   - **横ばい・変化が小さい軸**: あれば簡潔に触れる（例: 「伝達・構造化力はほぼ横ばい」）。
   - **低下した軸**: 直後が実施前より下がった軸があれば、1軸程度簡潔に記載（例: 「実現・ディレクション力は-0.33pt」）。
3. **バランス・総評**
   - 上記を踏まえた一言（例: 「全体的に向上し、とくに〇〇力が伸びている」「〇〇力は伸びたが、△△力は課題として残っている」等）。現状の「全体的に向上し、バランスが改善」に代えて、**その人ごとの数値に基づく表現**にする。

### 3.2 含めない／避けること

- 全員に同じ定型文のみを返さない（「全体的に向上し、バランスが改善」のみで終わらせない）。
- 直後の各軸の点数を**列挙するだけ**の説明にしない。比較・傾向を必ず入れる。

### 3.3 出力元・データ

- **入力**: 当該参加者の `pre_scores`（実施前5軸＋総合）、`post_scores`（直後5軸＋総合）、`skill_names`, `skill_keys`。
- **出力**: 上記 3.1 を満たす1文〜2文の傾向分析テキスト。改行は含めず、表セル用に `_cell_safe` でそのまま出力可能な形式。

---

## 4. C_block_2_1_body_P（実施前の説明）の扱い

- 現状: 「全体的に低め／標準的／高め」＋「特に〇〇力が低い／高い（X.XX点）」で、ある程度参加者ごとに差別化されている。
- 本要件では**変更不要**を原則とする。将来的に「直後との対比を匂わせる」表現を足す場合は、別要件で拡張する。

| 対象 | 内容 |

| **GASコード（個別）.gs** | 再生成により反映。 |

### 5.1 既存との整合

- **全体用**の `{{C_block_2_2_body}}` は、12_C_block_comment_requirements に従い別ロジック（全社集計）のままとする。本要件は**個別用**（_P 付き）のみを対象とする。
- 生成レポート（個別）.md の「レーダーチャートの説明」セクションも、同一ロジックで生成する場合は同じ文章が出力されることを想定する。

## 6. 受入基準（例）

- 参加者A: 実施前 2.20点 → 直後 2.93点、構想・コンセプト力 +1.00pt 等 → 「総合が+0.73pt向上。とくに構想・コンセプト力が+1.00pt伸びて3.00点に。バランスが改善している。」のような**数値に基づく傾向**が含まれる。
- 参加者B: 伝達・構造化力が横ばい、実現・ディレクション力が-0.33pt → その旨が簡潔に述べられ、参加者Aと**異なる文面**になる。
- 全参加者で「全体的に向上し、バランスが改善（…点、…点、…点）」の**同一文言のみ**になっていないこと。

## 8. C_block_2_3_body_P（1ヶ月後の説明）の仕様 ※Phase3のみ

### 8.1 設計方針（2026-03-11 改定）

**旧仕様（廃止）**: 全5軸のスコアを括弧内に列挙する形式（例: 「さらに向上し、バランスが改善（リサーチ・分析力2.33点、…）」）

**新仕様（現行）**: **数字を含めず、傾向を文章で描写する**。

> 理由: レーダーチャートの実際の数字は図表で確認できるため、説明欄は「何がどう変わったか」という傾向読解を担う。

### 8.2 文章構造

以下の順番で1〜2文を構成する。

| 順序 | 要素 | 例 |
|------|------|----|
| ① 全体傾向（直後比） | 1ヶ月後が直後と比べどう変化したか | 「直後からさらに定着が進んでいる。」「直後の水準を維持・微向上している。」「直後比でやや低下しているが、」 |
| ② 特筆すべき軸 | 特に伸びた軸 or 課題軸 | 「特にリサーチ・分析力の定着が顕著。」「構想・コンセプト力は継続的な実践が必要。」 |

### 8.3 全体傾向の判定基準（直後比）

| 直後比の変化量 | 文言 |
|--------------|------|
| +0.2pt 以上 | 直後からさらに定着が進んでいる。 |
| 0.0 〜 +0.2pt | 直後の水準を維持・微向上している。 |
| -0.2 〜 0.0pt | 直後と比べほぼ横ばいで安定している。 |
| -0.2pt 未満 | 直後比でやや低下しているが、 |

### 8.4 特筆すべき軸の判定基準（直後比）

- **定着が顕著**: 直後比 +0.2pt 以上の軸（最大2軸）→ 「特に〇〇・△△の定着が顕著。」
- **課題**: 直後比 -0.2pt 以下の軸（最大1軸）→ 「〇〇は継続的な実践が必要。」
- **変化が小さい（全軸 ±0.2pt 以内）** の場合:
  - 最高軸が 3.0pt 以上 → 「〇〇が強みとして定着している。」
  - 最低軸が 2.5pt 未満 → 「〇〇に伸びしろが残る。」

### 8.5 実装

| 項目 | 内容 |
|------|------|
| 関数 | `_generate_radar_description_follow_individual`（`report_generator.py`） |
| 引数 | `pre_scores`, `post_scores`, `follow_scores`, `pre_total`, `post_total`, `follow_total`, `skill_names`, `skill_keys` |
| 出力 | 数字を含まない傾向描写テキスト（改行なし、`_cell_safe` 適用済み） |
| 呼び出し元 | `generate_individual_slide_content_markdown` 内 |

### 8.6 出力例

| ケース | 出力テキスト |
|--------|------------|
| 定着が進んだ | 直後からさらに定着が進んでいる。特に具体化・検証力・実現・ディレクション力の定着が顕著。 |
| 維持・微向上 | 直後の水準を維持・微向上している。特にリサーチ・分析力の定着が顕著。具体化・検証力は継続的な実践が必要。 |
| 横ばい安定 | 直後と比べほぼ横ばいで安定している。リサーチ・分析力が強みとして定着している。 |
| やや低下 | 直後比でやや低下しているが、リサーチ・分析力は継続的な実践が必要。 |

### 8.7 未回答者の扱い

1ヶ月後CSVにデータがない参加者（follow_scores = None）は `-` を出力する（Phase3未回答者として扱う）。
