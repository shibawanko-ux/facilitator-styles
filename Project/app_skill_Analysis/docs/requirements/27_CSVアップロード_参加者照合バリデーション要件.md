# 27 CSVアップロード時の参加者照合バリデーション要件

CSVをアップロードした際に、直前フェーズのCSVと参加者（メールアドレス・氏名）が一致しているかを確認し、不一致の場合にユーザーへ警告する機能の要件を規定する。

---

## 1. 背景・課題

### 1.1 発生した問題

Phase3において、1ヶ月後.csv の参加者3名がメールアドレスのドメイン違いにより実施前.csv と紐づかず、1ヶ月後のスコアが「データなし（`-`）」として扱われる問題が発生した（2026-03-11 発覚）。

| 参加者 | 実施前.csv | 1ヶ月後.csv（問題） | 影響 |
|--------|-----------|------------------|------|
| 河辺　裕希 | yuki_kawabe@lifecard.co.jp | yuki_kawabe@aiful.co.jp | ドメイン違いで不一致 |
| 河合　俊亮 | syunsuke_kawai@aiful.co.jp | syunsuke_kawai@ag-design.biz | ドメイン違いで不一致 |
| 野田健太郎 | kentaro_noda@aiful.co.jp | kentaro_noda@ag-design.biz | ドメイン違いで不一致 |

### 1.2 やりたいこと

- CSVアップロード時に、直前フェーズのCSVと参加者を自動照合する
- 不一致（メールアドレスのドメイン違い・氏名の揺れ・参加者の過不足）を検出してユーザーに警告する
- 完全ブロックではなく「警告＋続行選択」とし、ユーザーが判断できるようにする

---

## 2. 照合対象フェーズの定義

| アップロードCSV | 照合対象（直前フェーズ） |
|---------------|----------------------|
| 直後.csv | 実施前.csv |
| 1ヶ月後.csv | 直後.csv（なければ実施前.csv） |
| 上長1ヶ月後.csv | 実施前.csv（対象者メールアドレス列で照合） |

---

## 3. 照合ロジック要件

### 3.1 メールアドレスの照合ルール

1. **完全一致**（正常）: そのままOK
2. **ローカル部分（@より前）が一致・ドメインが異なる**（警告）: 同一人物の可能性が高いとみなし、警告を表示しつつ続行可能
3. **ローカル部分も一致しない**（エラー警告）: 新規参加者または別人として警告

### 3.2 氏名の照合ルール

メールアドレス照合で紐づいた参加者について、氏名も確認する。

- **完全一致**: OK（スペース・全角半角の正規化後）
- **部分一致（姓のみ一致など）**: 注意として警告表示
- **不一致**: 警告表示（ただし続行可）

> **注意**: 氏名のスペース・全角半角の揺れ（例: 「新田善隆」と「新田　善隆」）は正規化して一致とみなす。

### 3.3 参加者数の差異チェック

- 直前フェーズより**参加者が少ない**: 「○名が未回答」として警告
- 直前フェーズより**参加者が多い**: 「○名が新規（直前フェーズに存在しない）」として警告

---

## 4. UIの要件

### 4.1 バリデーション実施タイミング

- CSVアップロード後、データ処理の**実行前**に照合を行い結果を表示する

### 4.2 照合結果の表示内容

以下を一覧で表示する：

```
【照合結果】直後.csv vs 実施前.csv

✅ 正常に照合できた参加者: 17名
⚠️  ドメインが異なる参加者: 2名
  - yuki_kawabe@lifecard.co.jp（実施前）← yuki_kawabe@aiful.co.jp（1ヶ月後）
  - syunsuke_kawai@aiful.co.jp（実施前）← syunsuke_kawai@ag-design.biz（1ヶ月後）
❌ 実施前.csv に存在しない参加者: 1名
  - unknown_person@example.com（新規 or 誤記）
⚠️  1ヶ月後.csv に未回答の参加者: 1名
  - kentaro_noda@aiful.co.jp（野田健太郎）

[このまま続行する]  [CSVを修正してアップロードし直す]
```

### 4.3 続行・中断の選択

- **「このまま続行する」**: ドメイン違いはローカル部分一致でマッチングし処理を継続。不一致者は「データなし（`-`）」として処理。
- **「CSVを修正してアップロードし直す」**: アップロード画面に戻る（処理はキャンセル）

### 4.4 警告レベルの定義

| レベル | 条件 | 表示 |
|--------|------|------|
| ✅ 正常 | 完全一致 | 緑色 |
| ⚠️ 注意 | ドメイン違いだが@前が一致 / 氏名の揺れ / 参加者数の過不足 | 黄色 |
| ❌ エラー警告 | @前も不一致（別人の可能性） | 赤色 |

## 5. 実装対象

| 対象 | 内容 |
|------|------|
| **バックエンド** | `src/csv_validator.py`（新規）に照合ロジックを実装 |
| **呼び出し元** | `app.py` のCSVアップロード処理ルート（Phase2・Phase3のCSV受け取り後） |
| **フロントエンド** | アップロード後に照合結果をモーダルまたはページ上部に表示。「続行」「戻る」ボタンを設置 |

### 5.1 `csv_validator.py` の主要関数

```python
def validate_csv_participants(
    upload_data: List[Dict],        # 今回アップロードしたCSVのデータ
    reference_data: List[Dict],     # 直前フェーズのCSVのデータ（照合対象）
    email_col: str = 'メールアドレス',
    name_col: str = '氏名',
) -> Dict:
    """
    Returns:
        {
            'matched': [...],          # 正常に照合できた参加者リスト
            'domain_mismatch': [...],  # ドメイン違い（@前一致）の参加者リスト
            'not_in_reference': [...], # 直前フェーズに存在しない参加者
            'missing_in_upload': [...],# 今回アップロードに存在しない参加者（未回答者）
            'name_mismatch': [...],    # 氏名不一致の参加者
        }
    """
```

## 6. 受入基準

- [ ] 直後.csvをアップロードすると実施前.csvとの照合結果が表示される
- [ ] ドメインが異なるが@前が一致する参加者が⚠️として一覧表示される
- [ ] 直前フェーズに存在しない参加者が❌として表示される
- [ ] 未回答者（直前フェーズにいるが今回CSVにいない）が⚠️として表示される
- [ ] 「続行」を選ぶとドメイン違いはローカル部分一致でマッチングして処理が進む
- [ ] 「戻る」を選ぶとCSVを再アップロードできる
- [ ] 全員が正常照合のときは警告なしで自動的に処理が進む（クリック不要）
