# 09 ブラウザで確認（表示）仕様

ダウンロード一覧の「ブラウザで確認」ボタンにより、各ファイルをブラウザ上で HTML として閲覧できるようにするための要件を定義する。

---

## 1. 目的とスコープ

- **目的**: ユーザーが「DL」ではなく「ブラウザで確認」を押したときに、レーダーチャート（画像）や表組み（CSV・Markdown 内テーブル）がその場で正しく表示されるようにする。
- **スコープ**: `/view/<path:filename>` エンドポイントによる表示、および Markdown レンダリング時の画像・表の扱い。

---

## 2. 対象ファイル種別と表示要件

| ファイル種別 | 拡張子 | 表示要件 |
|-------------|--------|----------|
| レーダーチャート（全体・個別） | .png | HTML ページ内で画像を表示する。画像は必ず表示され、ダウンロード用リンクも提供する。 |
| 生成レポート（Markdown） | .md | HTML に変換して表示する。**Markdown 内の画像参照**と**Markdown の表**を正しく HTML で表示する。 |
| スライド挿入内容（Markdown） | .md | 上記と同様。 |
| 個人別スコア推移・アンケート項目別平均比較表 等 | .csv | 表形式（HTML テーブル）で表示する。ヘッダー行を強調し、見やすく表示する。 |

---

## 3. 機能要件

### 3.1 PNG（レーダーチャート等）

- **FR-PNG-1**: 「ブラウザで確認」で開いたとき、PNG は **HTML ページ内に画像として表示**されること。
- **FR-PNG-2**: 画像は、データ URI（Base64 埋め込み）または同一オリジンで解決可能な URL により表示されること（外部サイトに依存しない）。
- **FR-PNG-3**: ページ内に「画像をダウンロード」など、元ファイルを DL するリンクを用意すること。
- **FR-PNG-4**: 日本語等を含むファイルパスでも、正しくファイルが開き画像が表示されること。

### 3.2 Markdown（生成レポート・スライド挿入内容）

- **FR-MD-1**: Markdown を HTML に変換し、見出し・リスト・太字・コードブロックが正しく表示されること。
- **FR-MD-2（表）**: Markdown の表記法（`| ... |` で囲まれた行）は **HTML の `<table>` としてレンダリング**され、ブラウザ上で表組みとして表示されること。
- **FR-MD-3（画像）**: Markdown 内の画像記法 `![](パス)` および `![alt](<パス>)` は **HTML の `<img>` に変換**されること。
- **FR-MD-4（画像の解決）**: 画像の `パス` が相対パスの場合、**現在表示している Markdown ファイルと同じディレクトリ**を基準に解決し、表示すること。これにより、個別レポート内のレーダーチャート画像がブラウザで確認画面に表示されること。
- **FR-MD-4a（画像の表示方式）**: 同じディレクトリに実体が存在する画像は **Data URI（Base64 埋め込み）** で `<img src="data:image/png;base64,...">` として埋め込むこと。これにより、別 HTTP リクエストや URL エンコード・プロジェクトフォルダ名の違いに依存せず、確実に表示される。同一ディレクトリ外へのパスは許可しない（パストラバーサル防止）。
- **FR-MD-4b（フォールバック）**: 画像ファイルが読み込めない場合のみ、`/view/` 用 URL（パーセントエンコード）を `src` に用いてよい。
- **FR-MD-5**: 画像が存在しない、またはアクセス不可の場合は、代替テキスト（alt）を表示するか、プレースホルダー表示とする。

### 3.3 CSV（個人別スコア推移・アンケート項目別平均比較表 等）

- **FR-CSV-1**: 「ブラウザで確認」で開いたとき、CSV は **HTML テーブル**として表示されること（1 行目をヘッダー、2 行目以降をデータ行）。
- **FR-CSV-2**: ヘッダー行は視覚的に区別できること（背景色・フォント等）。
- **FR-CSV-3**: 数値列は右揃えなど、可読性を考慮した表示とすること。
- **FR-CSV-4**: ページ内に「CSV ファイルをダウンロード」リンクを用意すること。
- **FR-CSV-5**: 日本語（ファイル名・セル内容）が文字化けせず表示されること（UTF-8 等）。

---

## 4. 非機能要件

- **NF-1**: `/view/` でアクセス可能なパスは、既存のセキュリティ方針（`_validate_file_access`）に従い、`projects` 配下の `reports`・`spreadsheet_export`・`uploads` に限定すること。
- **NF-2**: パスに含まれる日本語（例: プロジェクト名・ファイル名）が URL として正しく扱われ、表示・画像取得が失敗しないこと。実装では、Markdown 内の画像は同一ディレクトリのファイルを Base64 埋め込みで表示し、URL に依存しない。

---

## 5. 受け入れ条件（まとめ）

- ダウンロード一覧で「ブラウザで確認」をクリックしたとき、
  - **レーダーチャート（PNG）**: 画像が HTML ページ内に表示される。
  - **生成レポート（個別）Phase 2 (Markdown)**: 表組みが HTML の表として表示され、レーダーチャート画像が同じページ内に表示される。
  - **個人別スコア推移・アンケート項目別平均比較表（CSV）**: 表組みが HTML テーブルとして表示される。

---
