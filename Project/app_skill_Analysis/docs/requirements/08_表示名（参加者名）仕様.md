# 08 表示名（参加者名）仕様

個別レポート・スライド挿入内容・GAS で使用する**参加者表示名**の決定ロジックと、名前が影響する出力箇所を定義する。

---

## 1. 目的とスコープ

- **表示の一貫性**: レポート・スライド・CSV エクスポート・GAS で、同一参加者には同じ表示名を用いる。
- **CSV 氏名を優先**: フォームの「あなたの氏名を入力してください。」等で入力された氏名を正規化列 `氏名` として扱い、これを最優先で表示に用いる。
- **フォールバック**: いずれの CSV にも `氏名` が無い場合のみ、メールアドレスの @ より前の部分を表示名として用いる（ファイル名の一意性・スキップ防止のため）。

---

## 2. 表示名の決定ロジック

### 2.1 参加者表示名（個人別レポート・スライド・個別 GAS）

| 優先順位 | 取得元 | 説明 |
|----------|--------|------|
| 1 | 実施前.csv の `氏名` | 正規化後カラム `氏名` の値（空白除去後）。空でない場合に採用。 |
| 2 | 直後.csv の `氏名` | 同一メールアドレスでマッチした行の `氏名`。1 が空の場合に参照。 |
| 3 | 1ヶ月後.csv の `氏名` | 同上。1・2 が空の場合に参照。 |
| 4 | メールアドレスの @ より前 | 1〜3 がいずれも空の場合のフォールバック。一意な表示・ファイル名用。 |

- マッチングは **メールアドレス** をキーに行う。
- 列 `氏名` は、07_フォーム準拠_CSV取り込み仕様に従い「あなたの氏名を入力してください。」等から正規化される。

### 2.2 上長・評価者名（Phase 3 ギャップ分析等）

- 上長1ヶ月後.csv の **上長氏名**（または正規化で `上長氏名` にマッピングされた列）を優先し、無い場合は `氏名` を使用する。
- 実装: `report_generator.py` 等で evaluator / manager の `name` として参照。

---

## 3. 名前が影響する出力箇所

以下のいずれも、上記 2.1 で決まった **同一の表示名** を用いる。

### 3.1 生成レポート（個別）.md

| 箇所 | 形式例 | 説明 |
|------|--------|------|
| セクション見出し | `## N. {表示名}様 スキル分析（レーダーチャート）` | 参加者ブロックの見出し |
| 画像の代替テキスト | `![{表示名}様 スキル定着度推移](...)` | レーダーチャート画像の alt |
| 画像ファイル名 | `生成レポート（個別）_{プロジェクト名}_{表示名サニタイズ}_Phase{N}_レーダーチャート.png` | ファイル名に使用できない文字は `_` に置換 |

### 3.2 スライド挿入内容（個別）.md

| 箇所 | 形式例 | 説明 |
|------|--------|------|
| 参加者見出し | `## 参加者{N}: {表示名}` | 参加者ブロックのタイトル |
| プレースホルダー値 | `\| \`{{name_P}}\` \| {表示名} \|` | スライドに挿入する名前 |
| データソース表記 | `生成レポート（個別）.md（{表示名}様 > ...）` | 出典説明 |
| レーダーチャート画像参照 | 上記と同じファイル名 | 手動挿入用の参照ファイル名 |

### 3.3 個別 GAS（GASコード（個別）.gs）

- スライド挿入内容（個別）.md をパースして取得した **参加者名** と **各プレースホルダーの値** をそのまま `PARTICIPANTS_DATA` に埋め込む。
- `{{name_P}}` / `{{name}}` の値は、上記 md に記載された表示名（＝2.1 のロジックで決まった名前）となる。

### 3.4 エクスポート CSV

- 01_エグゼクティブサマリー.csv・03_ギャップ分析.csv・09_実施直後アクション項目.csv・10_1ヶ月後定着確認.csv 等の「氏名」列には、同じく 2.1 で決まった表示名を出力する。

### 3.5 個人別レーダーチャート PNG のファイル名（app.py）

- 表示名（2.1 で決定）をサニタイズした文字列をファイル名に使用する。表示名が空の場合は 2.1 のフォールバック（メール @ 前）を用い、レーダー生成をスキップしない。

---

## 4. 実装の責務

| 責務 | 配置 |
|------|------|
| 表示名の決定（実施前→直後→1ヶ月後の氏名、空ならメール @ 前） | `src/analyzer.py` の `analyze_individual_progress` |
| 個別レポート・スライド挿入内容での表示名の使用 | `src/report_generator.py`（`person['name']` を参照） |
| 個別レーダー PNG のファイル名・タイトル | `app.py`（`person.get('name')`、空ならメール @ 前でファイル名を生成） |
| 氏名列の正規化（フォーム質問文 → `氏名`） | `src/csv_normalizer.py` |

---

## 5. 他ドキュメントとの関係

- **07_フォーム準拠_CSV取り込み仕様**: 列 `氏名` のマッピング候補（「あなたの氏名を入力してください。」等）を定義。本 08 は「その `氏名` をどこで・どの順で参照し、表示にどう使うか」を定義する。
- **05_分析と出力仕様**: 個別レポート・スライド挿入内容・CSV は 05 の出力一覧に含まれる。それらの「参加者名」は本 08 の表示名を用いる。

---

## 6. まとめ

| 項目 | 内容 |
|------|------|
| 表示名の優先順 | 実施前 氏名 → 直後 氏名 → 1ヶ月後 氏名 → メール @ 前 |
| 一貫性 | レポート・スライド・GAS・CSV・PNG ファイル名で同一の表示名を使用 |
| フォールバック | 氏名が全て空の場合のみメール @ 前を使用（表示・ファイル名とも） |
