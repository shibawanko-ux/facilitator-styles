# 15 スライド3 組織別分析総評 文章差別化要件

スライド3（組織別分析）の「分析総評」である **強み**（`{{O_block_2_body}}`）と **伸びしろ**（`{{O_block_3_body}}`）について、組織ごとに数字以外も差別化された文章を構成するための要件を規定する。  
該当組織のメンバーの **活用意欲・アクション宣言・理解度** を必ず参照し、組織ごとに異なる文章になるようにする。

## 1. 目的

- 分析総評の強み・伸びしろが、組織を問わず「数字と引用するアクション宣言の文言だけが変わり、それ以外はほぼ同一」になっている状態を解消する。
- 該当組織に**関わるメンバー**の「活用意欲・アクション宣言・理解度」を**必ず確認**し、その組織の強み・伸びしろの**文章を組織ごとに構成し直す**。
- 読み手が「この組織はこういう意欲・理解度・宣言がある」と把握できるようにする。

## 2. 必須要件

| 要件ID | 内容 | データソース |
|--------|------|-------------|
| R1 | 組織ごとに、その組織のメンバー（post_data を所属部署でフィルタ）の **活用意欲（Q16A）** を集計し、強み・伸びしろのいずれかまたは両方のブロックに反映する（例：「○名中○名が活用意欲「湧いている」以上」）。**湧いている以上が0名の場合は、この一文は出力しない**（「意欲が高い」等と矛盾するため）。 | post_data、所属部署 |
| R2 | 強みブロックで引用するアクション宣言は、その組織のメンバーの Q17A のみから選び、強みスキル軸と関連するものを優先する。 | post_data、Q17A、所属部署、SKILL_TO_Q17A_KEYWORDS |
| R3 | 伸びしろブロックで引用するアクション宣言は、その組織のメンバーの Q17A のみから選び、弱みスキル軸の改善に資するものを優先する。強みブロックと**同じ宣言を繰り返さない**。 | 同上 |
| R4 | スキル軸の説明文について、組織の人数・スコア帯に応じた言い回しの切り替えを検討する（任意）。 | 02_組織別分析、人数、部署名 |
| R5 | **理解度**（WS理解度の組織別平均＝`{{OrkA_1}}` と同値）を分析総評の文章で**明示的に参照**する。例：「理解度は○.○点と高く、強みの定着が期待される」「理解度のさらなる向上のため、振り返りの機会があるとよい」。満足度と理解度は別々に閾値判定し、文言を切り替える。 | post_data、WS満足度、WS理解度、所属部署 |
| R6 | データ不足時（該当組織の Q16A/Q17A が無い、または人数0）は、スコアと固定説明のみで構成し、アクション宣言・活用意欲の文は省略する。エラーにせずフォールバック。 | — |

## 3. 推奨要件

| 要件ID | 内容 |
|--------|------|
| R7 | 組織の人数が1名・2名の場合は「参加者からは」「○名中○名が」など、人数が分かる表現にする。個人の特定は避ける。 |
| R8 | 強み・伸びしろで引用するアクション宣言が重複しないよう、強みで使用した Q17A を伸びしろ選定時に除外する。 |

## 4. 理解度の参照

- 組織別の **WS理解度の平均**（スライドの `{{OrkA_1}}` と同一算出）を、強み・伸びしろの文章で**必ず参照**する。
- **強みブロック**: 理解度が高い場合は「理解度は○.○点と高く、強みの定着が期待される」など。低い場合は「理解度のさらなる向上のため、振り返りの機会があるとよい」など。
- **伸びしろブロック**: 理解度に応じた締め（「理解度も高く、改善意欲がうかがえる」「理解度を高めるフォローがあると、伸びしろの補強につながる」等）。
- 満足度と理解度は別々に閾値判定してよい（両方高い／片方のみ高い／両方低めで文言を切り替える）。

## 5. データソース

- **post_data**（直後.csv）: `所属部署`、`Q16A`（活用意欲）、`Q17A`（アクション宣言）、`WS満足度`、`WS理解度`
- **組織別理解度**: 該当組織の WS理解度 の平均（＝`{{OrkA_1}}`）
- **02_組織別分析.csv** / dept_analysis: 部署名、人数、各軸スコア

## 6. 変更対象

- **report_generator_impl.py**: `extend_strength_text_impl` / `extend_weakness_text_impl` に組織別理解度・満足度・活用意欲を反映。`_select_action_declarations` に `exclude_action_texts` を追加。強みで使用した宣言を伸びしろで除外。
- **report_generator.py**: 組織別理解度（ork_a1）・満足度・活用意欲を算出して `extend_*` に渡す。強みで使用したアクション宣言を伸びしろ生成時に除外するため、`extend_strength_text_impl` の戻りで使用した宣言を取得し、`extend_weakness_text_impl` に `exclude_action_texts` で渡す。
- **GAS**: スライド挿入内容 Markdown をパースして ORGANIZATION_DATA を埋め込むため、Python 側で生成する Markdown が更新されれば、GAS を再生成することで新しい O_block_2_body / O_block_3_body が反映される。GAS 本体の置換ロジックは変更不要。
