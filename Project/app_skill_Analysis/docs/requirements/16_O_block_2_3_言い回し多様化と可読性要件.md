# 16 O_block_2_body / O_block_3_body 言い回し多様化と可読性要件

スライド3（組織別分析）の **強み**（`{{O_block_2_body}}`）と **伸びしろ**（`{{O_block_3_body}}`）について、**読み手がわかりやすい**ことと、**組織をまたいでもテンプレートで同じような文章に見えない**ことを両立するための要件を規定する。  
本要件は [15_スライド3_組織別分析総評_文章差別化要件.md](15_スライド3_組織別分析総評_文章差別化要件.md) を拡張し、表現のバリエーションと可読性のロジックを追加する。

---

## 1. 背景と課題

### 1.1 現状の課題

現状の O_block_2_body / O_block_3_body は、組織ごとに**数字・引用するアクション宣言・スキル名**は変わるが、以下の点で**言い回しがほぼ同一**であり、テンプレート感が強い。

- **スキル説明**: 「〇〇（X.XX点）は△△。」の形式が全組織で同一。説明文そのものもスキル軸ごとに1種類のみ。
- **活用意欲**: 「参加者○名中○名が活用意欲「湧いている」以上と回答し、現場での実践意欲が高い。」が固定。
- **アクション宣言の導入**: 強み・伸びしろそれぞれで固定の1文言のみ。
- **理解度の締め**: 閾値ごとに1文言のみ。

この結果、**読み手が「どの組織も同じ型で書かれている」と感じやすく、組織ごとの違いが伝わりにくい**。

### 1.2 目的

- **可読性**: 読み手が「何が強み／伸びしろか」「どの数字が根拠か」を把握しやすくする。
- **表現の多様化**: 組織インデックス・スコア帯・人数などに応じて**言い回しの候補を切り替える**ロジックを入れ、スライドをまたいでも「同じテンプレの繰り返し」に見えないようにする。
- **再現性**: 同じ入力データであれば同じ出力になるよう、選択ロジックは**組織インデックス**を主な選択子とする。

---

## 2. 読み手がわかりやすくする要件（可読性）

| 要件ID | 内容 |
|--------|------|
| K1 | 各ブロックは **「スキル評価 → 活用意欲／アクション → 理解度・締め」** の順で構成する。 |
| K2 | 1文は長くても2〜3節までを目安とし、スキル軸ごとに文を分ける現状を維持する。 |
| K3 | 数値（スコア・人数・理解度）は文中で明示する（R1・R5を維持）。 |
| K4 | スキル軸名は 02_評価フレームワーク に合わせる。 |
| K5 | データ不足時はスコアとスキル説明のみで短くまとめる。 |

---

## 3. 表現の多様化（テンプレートで同じ文章に見えないロジック）

組織ごとに**同じスロットでも異なる言い回し**が選ばれるようにする。選択は **組織の並び順（org_index: 0始まり）** を主な選択子とし、`org_index % 候補数` で再現可能に選ぶ。

### 3.1 スキル説明文のバリエーション（V1, V2, V3）

- **強み**: 各スキル軸ごとに2〜3種類の言い回しを定義。組織インデックスで選択。
- **伸びしろ**: 同様に各スキル軸ごとに2〜3種類の言い回しを定義。組織インデックスで選択。
- 言い回しは意味が同じで語順・語彙だけが異なる程度に留める。

### 3.2 活用意欲の言い回しバリエーション（V4, V5）

- 3種類以上の言い回しを定義。`org_index % 候補数` で選択。
- **湧いている以上が0名のときは、活用意欲の一文は出力しない**（「意欲が高い」等と矛盾するため）。
- 例（1名以上の場合のみ）:「参加者○名中○名が活用意欲「湧いている」以上と回答し、現場での実践意欲が高い。」「活用意欲は○名中○名が「湧いている」以上であり、学びを現場で活かす意欲がうかがえる。」「○名中○名が「湧いている」以上と回答しており、実践への意欲が高い。」

### 3.3 アクション宣言の導入文のバリエーション（V6〜V9）

- **強み**: 2種類以上の言い回し。組織インデックスで選択。
- **伸びしろ**: 2種類以上の言い回し。組織インデックスで選択。

### 3.4 理解度の締め文のバリエーション（V10, V11）

- 同一閾値・同一ブロック種別でも2種類以上の締め文を定義。組織インデックスで選択。

### 3.5 フォールバック締めのバリエーション（V12）

- 理解度なし時の締めも2種類以上用意し、組織インデックスで切り替える。

## 4. データソース・変更対象

### 4.1 データソース

- 既存と同様: post_data、02_組織別分析.csv、組織別スコア・人数。
- **追加**: 組織の表示順序のインデックス（org_index: 0始まり）。多様化の選択子として使用。

### 4.2 変更対象

| ファイル | 変更内容 |
|----------|----------|
| `src/report_generator_impl.py` | 活用意欲・アクション導入・理解度締め・フォールバック締め・スキル説明の**複数候補**を定義。`org_index` に基づいて選択するロジックを追加。`extend_strength_text_impl` / `extend_weakness_text_impl` の引数に `org_index: Optional[int] = None` を追加。 |
| `src/report_generator.py` | 組織ループで `org_index`（0始まり）を渡す。 |
| `src/analyzer.py` | 強み・弱みのスキル説明を**複数候補**で返すか、説明文選択を impl 側で行う。本実装では impl 側で複数候補を持ち org_index で選択する方式とする。 |
| 本要件ドキュメント | 15 を拡張。00_requirements_overview に 16 を追記。 |

### 4.3 既存要件との関係

- **15_スライド3_組織別分析総評_文章差別化要件**: R1〜R8 はそのまま満たす。
- **10_分析コメント品質要件**: 文章量（約300文字）・内容要素は維持する。

---

## 5. 受け入れ条件

- 複数組織のスライドを連続で見たとき、活用意欲・アクション導入・理解度締め・スキル説明の少なくともいずれかが組織ごとに**異なる言い回し**になっている。
- 同じプロジェクトを再生成した場合、**同じ O_block_2_body / O_block_3_body** が出力される（再現性）。
- R1〜R6 を満たしたままであること。
