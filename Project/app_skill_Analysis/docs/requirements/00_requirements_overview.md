# 要件定義 オーバービュー

実践スキル定着度分析アプリの要件を、目的からデータ・分析・出力・GAS連携まで一貫したロジックで整理したものです。アプリ実装の「正」は本 requirements フォルダのドキュメントとします。

## ドキュメント構成と流れ

```mermaid
flowchart LR
  subgraph input [入力・前提]
    A[01_目的と評価]
    B[02_評価フレームワーク]
    C[03_実施プロセス]
  end
  subgraph spec [仕様]
    D[04_データ仕様とフェーズ]
    E[05_分析と出力仕様]
    F[06_GAS連携]
    G[07_フォーム準拠CSV取り込み]
    H[08_表示名（参加者名）]
  end
  A --> B --> C --> D --> E --> F --> G --> H
```
