# 21 組織別スコアと所属部署の整合性

個別レポートと組織別スライドの数値が一致するための前提と、不一致が起きたときの原因・対策をまとめます。

---

## 1. 現象の要約

- **事象**: 組織別スライドの実施前スコア（Ogr*_1）が 0.00 になる、または個別レポートの該当者スコアと組織の数値が一致しない。
- **主因**: 実施前.csv と 直後.csv で「所属部署」の**文字列が一致していない**（末尾スペース、全角/半角の差など）。組織別は直後.csv の所属で org_list（02_組織別分析.csv）を決め、実施前スコアは `dept_analysis_pre.get(dept_name)` で取得するため、実施前側に同じキーが無いと 0.00 になる。

---

## 2. コード側の対策

- **所属の正規化（トリム）**: `src/analyzer.py` の `analyze_by_department` で、所属部署の値を `.strip()` してから集計キーに使用している。実施前・直後ともに同じ正規化がかかるため、「業務システム課」と「業務システム課 」は同一キーとして扱われる。
- **他で所属をキー・比較に使う場合**: 同様に `.strip()` を徹底する。Cursor ルール「組織別分析・所属部署ルール」を参照。

---

## 3. 運用側の対策

- **フォーム**: 実施前と直後で「所属部署」の選択肢を**完全に同一**にする（同じリストをコピーする、またはスプレッドシートで同じセル範囲を参照する）。選択肢の先頭・末尾にスペースを入れない。
- **レポート生成後**: 新規プロジェクトでは、`scripts/verify_organization_scores.py` を実行し、レベル A が全件OKであることを確認する習慣を推奨する。実行例: `python3 scripts/verify_organization_scores.py --project "プロジェクト名"`。出力はプロジェクトフォルダ直下の `組織別数値検証結果.md`。

---

## 4. 不一致が起きたときの確認手順

1. **実施前.csv の「所属部署」列**を開き、該当者の行の値を確認する（前後スペース・全角の有無）。
2. 直後.csv の所属と**完全に同じ文字列**になるよう、フォームの選択肢またはスプレッドシートの参照を修正し、必要なら CSV を再エクスポートして再アップロードする。
3. 検証スクリプトを再実行し、レベル A が OK になることを確認する。
4. それでも不一致が解消しない場合は、07_フォーム準拠 や `src/csv_normalizer.py` で所属の値に対する追加正規化（例: 全角統一）の検討をする。
