# 19 個別分析総評：アクション宣言連携と文章改善要件

個別レポート・個別スライド挿入内容の「分析総評」を、読み手がわかりやすくアクションしやすい文章にし、参加者自身の「アクション宣言」（Q17A）を反映するための要件を規定する。

---

## 1. 背景と課題

### 1.1 現状の課題

- **テンプレート化**: 分析総評の「改善・次のアクション」部分が、スキル名ごとの固定文言（例：「週1回の情報収集時間を設けることをお勧めします」「定期的なブレインストーミングセッションに参加することをお勧めします」）になっており、誰が読んでも同じような表現になりがちである。
- **曖昧な伝え方**: 「お勧めします」といった一般的な助言に留まり、**その人固有の意欲や宣言**が総評に活かされていない。
- **アクション宣言の未活用**: 直後アンケートの「アクション宣言」（Q17A）は「活用意欲・アクション宣言」セクションには記載されているが、**分析総評の文章作成時には参照されておらず**、総評と宣言が分離したままになっている。

### 1.2 目指す状態

- 分析総評が**参加者ごとに内容が差別化**され、読み手（本人・上司等）が「自分のこと」として理解しやすい。
- **アクション宣言（Q17A）を総評の材料として組み込み**、「宣言を踏まえたうえで、このスキルをこう伸ばすとよい」といった流れで書かれる。
- 改善提案が**具体的で実行しやすい表現**になり、「お勧めします」だけに頼らない。

## 3. 要件一覧

### R1：アクション宣言（Q17A）の参照を必須化

- 分析総評を生成する際、**当該参加者のQ17A（アクション宣言）を必ず入力として参照する**。
- Q17Aが空・未回答の場合は、後述のフォールバック（R4）に従う。
- 参照元データ: 直後.csv（post_data）の当該参加者行の「Q17A」「アクション宣言」「Q17A: アクション宣言」等の列。

### R2：総評の「改善・次のアクション」部分でアクション宣言を活かす

- **弱みスキルに対する改善・次のアクション**を述べる段落で、次のいずれか（または両方）を行う。
  - **パターンA**: 参加者のアクション宣言を要約・引用し、「〇〇という宣言があり、この意欲を［弱みスキル名］の向上につなげるとよいでしょう」のように、宣言と弱みスキルの結びつけを明示する。
  - **パターンB**: アクション宣言の内容が弱みスキル（リサーチ・分析力、構想・コンセプト力等）の補強に直結する場合、「宣言にあるように〇〇を実践すると、［弱みスキル名］の向上に役立ちます」のように、宣言を具体的なアクションとして総評に組み込む。
- テンプレート的な「〜をお勧めします」のみで終わらせず、**その人の宣言に言及したうえで**、必要なら補足の一言を添える形とする。

### R2.5：変更後の具体的行動提案（要約）の候補と選択

- **候補の用意**: 各弱みスキル（リサーチ・分析力、構想・コンセプト力、具体化・検証力、伝達・構造化力、実現・ディレクション力）ごとに、「変更後の具体的行動提案」用の候補を**5つ程度**用意する。
- **相性の付与**: 各候補には、**その強みを持つ人に特に有効**であることを示す「相性」（強みスキル名。例：`pairs_with: 'リサーチ・分析力'`）を付与できる。誰にでも使える汎用候補は相性なし（`pairs_with: None`）とする。
- **1つだけ選択して表示**: 総評に表示する際は、対象者の**強み**（スコアが最も高いスキル）に合う候補（相性が一致するもの）を優先して**1つだけ**選択し、「加えて、〇〇」の形で分析総評に追記する。対象者にとって行動するとポイントが上がりやすい提案を選ぶ意図とする。
- **フォールバック**: 強みに合う候補が存在しない場合は、汎用候補（相性なし）を1つ選んで表示する。

### R3：汎用「お勧めします」文言の見直し

- スキル名ごとの**固定テンプレート文言**（例：「週1回の情報収集時間を設けることをお勧めします」「定期的なブレインストーミングセッションに参加することをお勧めします」「プレゼンテーションの機会を増やすことをお勧めします」「リーダーシップの機会を積極的に取り入れることをお勧めします」）は、**第一候補としては使わない**。
- 優先順位:  
  1. 当該参加者のQ17Aを要約・引用し、弱みスキルとの関連を示す文。  
  2. Q17Aが空または総評に組み込みにくい場合のみ、スキルに応じた**短く具体的な**補足（例:「まずは小さく試すサイクルを回すと、検証力の定着につながります」）を用いる。  
  3. それでも表現が必要な場合に限り、既存の「お勧めします」系を短く用いてもよいが、**曖昧で長い定型文は避ける**。

### R4：Q17Aが空・未回答の場合のフォールバック

- Q17Aが取得できない、または空の場合は、次のようにする。
  - 強み・スコア変化の記述は現行どおり行う。
  - 「改善・次のアクション」部分では、**その人向けの汎用テンプレートに依存しすぎない**短い一文（例:「［弱みスキル名］では、日々の業務で〇〇を意識すると伸びやすくなります」）に留めるか、あるいは「今後のアクション宣言を踏まえたフォローが有効です」のような、読み手が「次の一手」を考えやすい表現にする。
- フォールバック時も「週1回の情報収集時間を…」「定期的なブレインストーミング…」等の**長い定型文をそのまま並べない**。

### R5：読み手がわかりやすく・アクションしやすい文章

- **主語・目的を明確に**: 「誰が」「何をするとよいか」が伝わるようにする。
- **次の一手がイメージできる**: 抽象的な「お勧めします」ではなく、「まず〇〇から始めるとよい」「〇〇の機会を1回でも設けると」など、具体的な一歩が浮かぶ表現を心がける。
- **重複の削減**: レーダーチャートの説明やスコア推移表と完全に同じ文言の繰り返しを避け、総評では「解釈と次のアクション」に集中する。

### R6：活用意欲（Q16A）の扱い（任意）

- 分析総評のトーンや文末を、活用意欲（Q16A）が「湧いている」等の場合は前向きに、「どちらともいえない」等の場合は「まずは小さく始める」ような表現に寄せることは**任意**とする。優先はR1〜R5。

## 5. 修正ポイント例（要件に沿ったBefore/Afterイメージ）

### 例1：リサーチ・分析力が弱みの場合

- **Before（現状）**: 「リサーチ・分析力は0.67ポイント向上していますが、さらなる向上のため、より深い分析を行うために、週1回の情報収集時間を設けることをお勧めします。」
- **After（イメージ）**: 参加者のアクション宣言が「各サービスを利用するユーザーの文字に表せない意思を汲み取る」の場合  
  「リサーチ・分析力は0.67ポイント向上しています。『ユーザーの意思を汲み取る』という宣言を、日々のヒアリングや振り返りで意識すると、さらに深い分析につながります。」

### 例2：構想・コンセプト力が弱みの場合

- **Before（現状）**: 「アイデアを広げるために、定期的なブレインストーミングセッションに参加することをお勧めします。」
- **After（イメージ）**: 参加者のアクション宣言が「答えを誘導するような質問をしない」の場合  
  「構想・コンセプト力の向上には、『誘導しない質問』を意識した対話が有効です。宣言にあるように、日々の打ち合わせで問いの立て方を少し変えるだけでも効果が期待できます。」

### 例3：伝達・構造化力が弱みの場合

- **Before（現状）**: 「情報を整理して伝えるために、プレゼンテーションの機会を増やすことをお勧めします。」
- **After（イメージ）**: 参加者のアクション宣言が「収集した情報を分析する際にグループ分けをして整理し、言語化して相手に伝えられるようにする」の場合  
  「伝達・構造化力は、宣言にある『グループ分けして整理し、言語化して伝える』を、会議の前後や報告の場で一度試すところから始めると、定着しやすくなります。」

## 6. 受け入れ基準

- 生成レポート（個別）の各参加者「分析総評」に、**当該参加者のQ17A（アクション宣言）が空でない限り、その内容を反映した記述が含まれる**こと。
- 改善・次のアクションの部分で、**スキル名ごとの長い定型「お勧めします」のみが並ぶ状態が解消**されていること。
- **変更後の具体的行動提案**は、各弱みスキルごとに5つ程度の候補から、**対象者の強みに合う1つ**（または汎用候補）だけが「加えて、〇〇」の形で表示されていること（R2.5）。
- 読み手が「何をすればよいか」を想像しやすく、**アクションしやすい文章**になっていること（品質はサンプル確認で判断）。

## 9. 文章構造の改定仕様（2026-03-11 改定）

### 9.1 改定の背景

2026-03 の実装で、`_generate_analysis_summary` の文章全体を**数字なし・定性描写型**に刷新した。
改定前は Section 5（改善・次のアクション）の弱みスコアや変化量に数字が含まれ、読み手に数字の印象が残りやすかった。改定後は感覚的に伝わる定性表現で統一し、読みやすさを向上させた。

### 9.2 `_generate_analysis_summary` の文章構造（現行）

文章は **3つのまとまり** で構成される。各まとまり内は接続助詞でつなぎ、文末の「〜ています。」が連続しないようにする。

| まとまり | 構成要素 | 出力例 |
|---------|---------|--------|
| **文1: 変容＋強み** | 変容の理解（接続助詞で終わる節）＋強みの認識（文末で締める） | 「研修効果が現場に着実に定着しており、特にリサーチ・分析力が大きく伸びて強みとなっています。」 |
| **文2: 上長評価＋ギャップ**（Phase3のみ） | 上長コメント（「もあり、」で接続）＋ギャップの示唆（文末で締める） | 「上長からは『〇〇』という評価もあり、〇〇力は自己評価が高く自信を持って取り組めています。」 |
| **文3: 改善・次のアクション** | 弱みスキル＋Q17A引用または短いフォールバック＋行動提案1文 | 「〇〇力は引き続き成長の余地があります。『宣言』を向上につなげるとよいでしょう。加えて、〇〇。」 |

> **接続ルール**: 各まとまり内の要素が片方しかない場合は単独で文として出力する（文末を「。」で締める）。

### 9.3 Section 1：変容の理解の判定基準

| フェーズ | 条件 | 文言 |
|---------|------|------|
| Phase3 | follow_total - pre_total > 0.5 | 「研修効果が現場に着実に定着しています。」 |
| Phase3 | > 0.1 | 「実践を通じて着実な成長が見られます。」 |
| Phase3 | ≥ -0.1 | 「研修直後の水準を維持しながら実践を継続しています。」 |
| Phase3 | < -0.1 | 「継続的な実践が次の伸びにつながります。」 |
| Phase2 | post_total - pre_total > 0.5 | 「研修を通じて大きな成長が確認できました。」 |
| Phase2 | > 0 | 「研修を通じて着実な成長が見られます。」 |
| Phase2 | ≤ 0 | 「研修前後の変化は限定的でしたが、学びの基盤は築かれています。」 |

### 9.4 Section 2：強みの認識の判定基準

- 最終スコア（follow → post → pre の優先順）の最高軸を強みとする。
- 実施前比 +0.5pt 超 → 「大きく伸び、強みとなっています。」
- それ以外 → 「強みとして定着しています。」
- スコアの数字は含めない。

### 9.5 Section 5：改善・次のアクションの構成（改定後）

```
[弱みスキル名]は引き続き成長の余地があります。（または「着実に伸びていますが、さらに定着させるために、」）
[Q17A引用文 or フォールバック一文]
加えて、[行動提案1文]
```

#### 弱み判定の分岐

| 条件 | 文言（弱み説明部） |
|------|-----------------|
| 最終スコア - 実施前スコア < 0.5 | `{弱みスキル名}は引き続き成長の余地があります。` |
| ≥ 0.5 | `{弱みスキル名}は着実に伸びていますが、さらに定着させるために、` |

> 改定前は変化量の数値（例: `0.67ポイント向上していますが`）を含んでいたが、**数字なし**に変更した。

#### Q17A引用（R2 準拠）

- Q17A が 50 文字以下 → そのまま引用。
- Q17A が 50 文字超 → 先頭 47 文字 + 「…」に截断。
- 引用文: `「{quote}」という宣言を、{弱みスキル名}の向上につなげるとよいでしょう。`

#### Q17A空時のフォールバック（R4 準拠・改定後）

旧仕様の `followup_fallback`（長い汎用文）を廃止し、以下の短い定性一文のみを使用する。

| 弱みスキル | フォールバック文 |
|-----------|----------------|
| リサーチ・分析力 | 日々のヒアリングや振り返りで「深掘り」を1つ意識すると、分析力の定着につながります。 |
| 構想・コンセプト力 | 日々の打ち合わせで問いの立て方を少し変えるだけでも、コンセプト力の土台が育ちます。 |
| 具体化・検証力 | 小さく試すサイクルを1回でも回すと、検証力の定着につながります。 |
| 伝達・構造化力 | 報告や共有の場で「グループ分けして伝える」を一度試すと効果的です。 |
| 実現・ディレクション力 | チームの場で役割を一つ決めて進める機会を設けると、実践につながります。 |
| （その他） | `{弱みスキル名}では、日々の業務で小さく試す機会を設けると伸びやすくなります。` |

#### 行動提案候補（R2.5 準拠）

`CONCRETE_ACTION_CANDIDATES` として各弱みスキル5候補を定義。
候補には `pairs_with`（その強みを持つ人に特に有効な強みスキル名 or `None`）を付与。
対象者の強み（最高スキル）に `pairs_with` が一致する候補を優先し1つ選択。
なければ `pairs_with: None`（汎用）を選択。
表示形式: `加えて、{selected_text}`
