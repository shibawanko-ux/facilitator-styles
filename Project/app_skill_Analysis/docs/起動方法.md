# 起動方法

## 必要なもの

- Python 3.8 以上
- ターミナル

---

## 手順

### 1. フォルダに移動する

ターミナルの**今いる場所**に合わせて、次のどちらかを実行します。

- **プロンプトに `14_Workspace` と出ているとき**（ワークスペースルートにいる）  
  → `Project/app_skill_Analysis` に移動します。

```bash
cd Project/app_skill_Analysis
```

- **プロンプトに `app_Facilitator_type_diagnosis` など、別の app フォルダ名が出ているとき**  
  → `Project/` 内の兄弟フォルダなので、ひとつ上に戻ってから `app_skill_Analysis` に入ります。

```bash
cd ../app_skill_Analysis
```

移動できたかは、`ls` で `app.py` と `requirements.txt` が見えればOKです。

---

### 2. ライブラリを入れる

```bash
pip3 install -r requirements.txt
```

---

### 3. アプリを起動する

```bash
python3 app.py
```

ターミナルに「ブラウザで … にアクセスしてください」と**ポート番号付きのURL**が表示されたら起動できています。

---

### 4. ブラウザで開く

ターミナルに表示されたURLを、そのままブラウザのアドレス欄に入力して開きます。

- 通常は **http://127.0.0.1:5001** または **http://localhost:5001** です（ポートは **5001**）。
- **表示されているURLをそのまま使う**のが確実です。

---

### ブラウザに何も表示されない・403 になる場合

1. **アプリが動いているか確認する**  
   手順3で起動したターミナルはそのままにし、**別のターミナル**を開いて次を実行します（ポートはターミナルに表示された番号に合わせる）。  
   `curl http://127.0.0.1:5001/ping`  
   「OK」と表示されれば、アプリは動いています。

2. **動いているとき**  
   ブラウザでターミナルに表示されたURL（例: http://127.0.0.1:5001）を開き直してください。シークレット（プライベート）ウィンドウで試してもよいです。

3. **「OK」も出ないとき**  
   手順3のターミナルでエラーが出ていないか確認し、もう一度 `python3 app.py` からやり直してください。

---

## うまくいかないとき

| 出たメッセージ | 対処 |
|----------------|------|
| `no such file or directory` | 今いる場所を確認。**ワークスペースルートにいるとき**は `cd Project/app_skill_Analysis`。**Project 内の別の app にいるとき**は `cd ../app_skill_Analysis`。 |
| `command not found: python` | `python3` と `pip3` を使ってください。 |
| `requirements.txt` がない | 手順1で Project/app_skill_Analysis に移動できていません。プロンプトのフォルダ名を確認し、上記のどちらの `cd` かを使い分けてやり直し。 |
| **Port 5000 is in use**（ポート5000が使用中） | このアプリは標準で**ポート5001**を使うようにしてあります。もう一度 `python3 app.py` を実行し、表示されたURL（例: http://127.0.0.1:5001）をブラウザで開いてください。 |
| 403 アクセスが拒否されました | ターミナルに表示されたURL（**http://127.0.0.1:5001** など）で開き直す。別ターミナルで `curl http://127.0.0.1:5001/ping` を実行し「OK」が出るか確認。 |
| ページが表示されない | 手順3のターミナルを閉れていないか確認。表示された**ポート番号**（5001など）のURLを、そのままブラウザで開く。 |

## GASコード・レポートでよくある問題

### Phase 2用 GASコードのボタンが表示されない

**「Phase 2用 GASコードを生成」ボタンは、実施前・直後のCSVがあるかではなく、「Phase 2 のレポートが既に生成されているか」で表示が決まります。** 実施前・直後のCSVをアップロードしただけではボタンは出ません。一度**「分析実行」**を行い、Phase 2 のレポートが `reports/` に生成されている必要があります。

| パターン | 対処 |
|----------|------|
| 分析未実行 | そのプロジェクトを選択した状態で **「分析実行」** を行う。 |
| 別プロジェクトを選択している | 「GASコードを生成」を開く前に、目的のプロジェクトが選択されているか確認する。 |
| 上記でも出ない | ページを再読み込みしてから再度プロジェクトを選択し、「GASコードを生成」を開く。開発者ツール（F12）のネットワークで `files-status` のレスポンスに `max_phase: 2` が含まれるか確認する。 |

### 活用意欲・アクション宣言（Q16A・Q17A）が表示されない

生成レポート（個別）の「活用意欲・アクション宣言」や 09_実施直後アクション項目.csv の Q16A・Q17A 列がすべて「-」や空欄になる場合、直後.csv の列名がアプリのマッピング（`src/csv_normalizer.py`）で正しく Q16A・Q17A に対応していない可能性があります。本アプリでは Q16A・Q17A を Q1 より前に評価する順序でマッピングしており、質問文そのものも候補に含めています。**修正後は「Phase 2 分析」を再実行**すると、09 CSV と生成レポート（個別）に正しく値が入ります。

### 組織別スライドの強み・弱みが更新されない

Python で再生成したスライド挿入内容や GAS コードは新しい強み・弱みになっているのに、Google スライド上で GAS を実行しても組織別スライドの強み・弱みが前回のままになる場合、GAS は「プレースホルダー文字列」を探して置換する実装のため、2回目以降はスライド上にプレースホルダーが残っておらず置換されないことが原因です。**対処**: (1) プロジェクトで `scripts/regenerate_project_reports.py` を実行して GAS コードを再生成する。(2) 再生成した GAS の全文を Google スライドに紐づいた Apps Script エディタにコピーして上書き保存する。(3) スライドで GAS のメニューから「データをスライドに反映」等のメイン処理を再実行する。これで強み・弱みが今回のデータに更新されます。

## 補足

- 実施前のCSVだけアップロードした場合はレポートは出ません。直後・1ヶ月後のCSVもアップロードすると分析結果をダウンロードできます。
