# クライアント向けワークショップ振り返り資料の作り方（手順）

毎回このファイルを見れば手順が分かります。

**チャットで「始めましょう」と入力すると、この手順に従って一つずつアテンドしながら進めます。** 各ステップで何をすべきか案内し、あなたの入力や確認を待ってから次に進みます。

---

## 1. プロジェクトを決める

- **新規**: プロジェクト名を入力する（例: `20260130_アイフル`）。`projects/` 配下に同じ名前のフォルダが作られる。
- **続き**: 既存のプロジェクトを一覧から選ぶ。

---

## 2. 入力する

**一つずつ**入力し、送信するたびに結果を出力する。全部まとめてではなく、以下の順で進める。

### 2.1 チームごとに Good と More を入れる

- **Aチーム** … Good の内容と More の内容を入れる → 送信 → 結果を出力する。
- **Bチーム** … Good の内容と More の内容を入れる → 送信 → 結果を出力する。
- **Cチーム以降** … 同様に、チームごとに Good と More を入れて送信 → 結果を出力する。チームがなくなるまで繰り返す。

すべてのチームが終わったら「**次へ**」と入力する。

### 2.2 印象が強いメンバーを入れる（「次へ」のあと）

- **誰**（名前）と**内容**を箇条書きで入れる（印象に残った理由・行動・関わり方など、事実ベース）。
- 送信 → 結果を出力する。

終わったら「**次へ**」と入力する。

### 2.3 各ファシリテーターの所感を入れる（「次へ」のあと）

- **名前**と**内容**を箇条書きで入れる（メイン・サブごとに、その人の所感をそのまま）。
- 送信 → 結果を出力する。

ここまで終わったら、次のステップ「3. ①〜⑦を生成する」に進む。

---

必要なら「全体傾向（④）」「組織課題仮説（⑤）を含める」をオンにする（入力の前か、生成の前に指定）。

---

## 3. ①〜⑦を生成する

アプリで **①→②→③→④→⑤→⑥→⑦** の順に編集・生成する。結果はそのプロジェクトの `output/` に保存される。

各ステップでやること：

- **①** … ファシリテーターから見た Good & More を整理する。Good は「〜ができていた」系の要約タイトル＋内容、More は「〜する」「〜を意識しよう」系の要約タイトル＋内容。チーム別に出力。
- **②** … 印象が強いメンバーを整理する。印象に残った理由、行動・姿勢・関わり方（事実ベースのみ。人物評価・性格判断はしない）。ワークショップで特に良かった人なので、各人について応援や「〜してほしい」で背中を押す文章で締める。
- **③** … 各ファシリテーターの所感を整理する。ファシリごとに分けて、言い回し・温度感をそのまま保つ。統合・要約しない。
- **④** … ワークショップ後の全体傾向をまとめる（含める場合のみ）。Good と More をそれぞれ3項目・50文字程度。「〜傾向が見られた」を基本表現にする。
- **⑤** … 見えた組織課題仮説をまとめる（含める場合のみ）。タイトル20文字以内＋リード文50文字以内、最大3項目。「〜可能性がある」「〜傾向が見られた」に留める。
- **⑥** … まとめを作る。リード文は全体総評として結論づける文章（150文字程度、最後に「次のステップが実務につながる」文脈）。要素3点は**今後やったほうがよい3点**を箇条書きで記載する（各20文字程度）。さらに **総合のGood＆More**（①を総合した Good 3条・More 3条、各80文字程度）と **見えた組織課題仮説**（全入力から3項目、タイトル＋説明。「〜傾向が見られた」に留める）を P3 用に含める。
- **⑦** … 今回のワークショップ総括の文言を入れる。実施WS種別に応じた固定文言（アプリが自動で選ぶ）。変更しない。

補足：役割の指示は不要（アプリが毎回つける）。④・⑤は「含める」をオンにした場合だけ生成する。

---

## 4. ⑨を作る場合だけ

- **⑧** の4項目（なぜこの支援プログラムを選んだか／直前に実施したWS／参加者にどんな続きの体験をしてほしいか／気になっている点）を入力する。推薦したいプログラム名（例: イチオシ：インタビュー Workshop）をあわせて指定すると、それを「おすすめワークショップ」として⑨に反映される。
- 4つ揃ったら **⑨** を生成する。結果は `output/09_support_program.md` に保存（全体のタイトル・全体サマリー・傾向と課題仮説・おすすめワークショップを含む）。
- ⑨を作らない場合は ⑧・⑨はスキップしてよい。

---

## 5. スライドに貼る

`projects/プロジェクト名/output/` の各ファイルの文を、サンプルと同じ構成のスライドにコピーする。

| 貼る場所 | 元ファイル |
|----------|-------------|
| **P3** | 07_overview.md（総括）、04_trends.md（全体傾向）、06_summary.md（まとめ）、05_hypothesis.md（組織課題仮説） |
| **P9〜P12** | 01_good_more.md（チーム別 Good＆More） |
| **P13** | 02_members.md（印象が強いメンバー） |
| **P14** | 03_impressions.md（各ファシリの所感） |
| **P15** | 09_support_program.md（全体のタイトル・各種支援プログラム文） |

---

## 6. 手作業で仕上げる

次のスライドはアプリでは作らないので、手で編集する。

- 表紙（クライアント名・日付など）
- Agenda
- 参加者（座席表）
- Appendix（ワークショップメニューなど）
