# プロジェクト（レポート）格納フォルダ

1回のワークショップ振り返りレポートを**1プロジェクト**とし、プロジェクトごとにこの配下にフォルダを作成して、入力・生成アウトプットを格納する。

## 利用フロー（プロジェクト名から始める）

1. **プロジェクト名の入力** … 新規のときはプロジェクト名（例: `20260130_アイフル`）を入力。既存を続きから編集する場合は一覧から選択。
2. **プロジェクトフォルダの作成 or 選択** … 入力された名前でここにフォルダを作成する（新規）、または既存フォルダを選択する。
3. **入力** … そのプロジェクトに紐づけて Good/More・印象が強いメンバー・各ファシリの所感を入力。必要なら `input/` に保存。
4. **生成** … ①〜⑨の順で編集・生成し、結果をそのプロジェクトの `output/` に格納する。

## プロジェクトフォルダの命名規則

- 推奨: `YYYYMMDD_クライアント名` または `YYYYMMDD_クライアント名_WS名`
- 例: `20260130_アイフル`、`20260225_○○社_ResearchProcess`

## 各プロジェクトフォルダの中身

- **input/**（任意） … 入力テキスト（Good/More、印象が強いメンバー、各ファシリの所感）をファイルで保存する場合に使用。
- **output/** … 生成した ①〜⑨ のセクション別テキストを格納。スライド（P3, P9〜P15）に貼る文言の出所。

詳細は [docs/02_data_design.md](../docs/02_data_design.md) を参照。
