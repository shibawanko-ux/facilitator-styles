# システムコーチング® プランニング資料生成ツール

ORSC®（Organization & Relationship Systems Coaching®）のインタビュー内容を分析し、
コーチングプランニング資料のテキストをAIで自動生成するCLIツールです。

---

## フォルダ構成

```
app_SystemCoaching_planner/
├── analyze.py              # メインスクリプト
├── README.md               # この使い方ガイド
├── requirements.txt        # 依存パッケージ
├── .env                    # APIキー設定（要編集）
├── prompts/
│   └── system_prompt.txt   # Claude用プロンプト（ORSCフレームワーク）
├── clients/                # クライアントごとのフォルダ
│   └── クライアント名/
│       ├── info.json       # 基本情報（初回入力で自動保存）
│       ├── interview_01.txt # インタビューテキスト
│       ├── session_01.m4a  # 音声ファイル（保管用）
│       └── output/
│           └── YYYY-MM-DD_コーチングプラン_クライアント名.md
└── venv/                   # 仮想環境
```

---

## セットアップ（初回のみ）

### 1. APIキーを設定する

`.env` ファイルを開いて、Anthropic の API キーを設定します。

```
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxx
```

### 2. 依存パッケージをインストールする（未済の場合）

```bash
cd Project/app_SystemCoaching_planner
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
```

---

## 使い方

### 1. インタビューテキストを用意する

音声データを別途テキストに書き起こし、以下のどちらかで準備します。

**方法A（推奨）: テキストファイルをクライアントフォルダに置く**

```
clients/
└── 松岡家/
    └── interview_01.txt   ← ここに置く
```

**方法B: 実行時に直接貼り付ける**

実行中にターミナルへ貼り付けて「END」で終了します。
貼り付けたテキストはそのままファイル保存もできます。

---

### 2. スクリプトを実行する

```bash
cd Project/app_SystemCoaching_planner
./venv/bin/python analyze.py
```

---

### 3. 対話形式で入力する

実行すると順番に選択・入力を求められます。

#### 既存クライアントがある場合

```
--- クライアント / 組織・チームを選択 ---
  既存のクライアント:
    1. 松岡家
    2. ○○株式会社
    3. 新規作成

  番号を入力してください（1〜3）: 1

  保存済みの基本情報:
    対象者:       松岡家 兄弟3名
    コーチ:       Takuya
    期間:         2023年10月 〜 2024年2月
    セッション数: 5回
    ツール:       Zoom、Googleツール

  この情報を使いますか？（Enterでそのまま使う / u で更新）:
```

- **Enterキー**: 保存済みの情報をそのまま使う（2回目以降はここだけ）
- **u を入力**: ヒアリング形式で基本情報を再入力・上書き保存

#### 新規クライアントの場合

「新規作成」を選ぶとヒアリングが始まります。

```
  新規クライアントのヒアリングを始めます。
  ------------------------------------------------
  Q1. 対象の組織・チーム・クライアントの名前を教えてください: 松岡家
  Q2. 資料に記載する正式な対象者名を教えてください
      （例: 松岡家 3名 / 松岡家 チーム）: 松岡家 兄弟3名
  Q3. コーチの名前を教えてください: Takuya
  Q4. セッションはいつ頃から始まりますか？（例: 2026年4月）: 2023年10月
  Q5. 終了予定はいつ頃ですか？（例: 2026年9月）: 2024年2月
  Q6. セッションは何回を想定していますか？（デフォルト: 5）:
  Q7. 利用ツールを教えてください（デフォルト: Zoom、Googleツール）:

  ヒアリング完了！
  → フォルダ作成:  clients/松岡家/
  → 基本情報保存:  clients/松岡家/info.json
```

---

### 4. 出力ファイルを確認する

分析が完了すると、以下の場所にMarkdownファイルが生成されます。

```
clients/松岡家/output/2026-03-16_コーチングプラン_松岡家.md
```

#### 生成される内容

| セクション | 内容 |
|---|---|
| コーチングの目的と体制 | 対象・期間・コーチ・目的・目指す状態 |
| システムの現状/願い | ありたい夢・強み・コーチ所感・最悪な夢 |
| スケジュール / ロードマップ | Step0〜5 + 各セッションのワーク内容 |

## 音声ファイルの管理

音声ファイルはクライアントフォルダにそのまま置いておけます。
スクリプトは `.txt` `.md` のみを読み込み対象とするため、音声ファイルは影響しません。

```
clients/松岡家/
├── session_01_2023-10.m4a   # 第1回音声
├── session_02_2023-11.m4a   # 第2回音声
├── interview_01.txt          # 第1回 文字起こし
├── interview_02.txt          # 第2回 文字起こし
└── output/
```

## よくある操作

### 新しいクライアントを追加する

```bash
./venv/bin/python analyze.py
# → 「新規作成」を選んでクライアント名を入力するだけ
```

### 基本情報を更新する

```bash
./venv/bin/python analyze.py
# → 「n で再入力」を選択して上書き保存
```

### 複数回のインタビューを統合して分析する

複数の文字起こしテキストを1つの `.txt` にまとめてからフォルダに置いてください。
