# 改善案：プロジェクト1 A5〜A7（要求分析&仕様化）

## ■ A5｜ユーザー体験の構想・提案能力

### 【負のサイクルの可視化】
・利用状況把握で得たインサイトをもとに、筆記具を使う場面においてユーザーがどこで課題に直面し、それがどのように連鎖・悪化するかを「負のサイクル」として可視化した。
・筆記具は「使おうと意識して使うもの」ではなく、「業務中に突発的に気づいた課題が連鎖するタイミングで反射的に手が伸びるもの」という利用実態が明らかになっていたため、その瞬間にどんな体験上の摩擦が生じているかを起点に整理した。
・「どこで詰まるか」だけでなく、「なぜそこで詰まりやすくなっているか」という構造を明らかにすることで、表面的な不満の裏にある体験上の本質的な問題を特定した。

### 【ユーザーストーリーの作成】
・特定した課題構造をもとに、「新しい筆記具がどんな体験を実現すべきか」という観点でユーザーストーリーを定義した。
・「誰が・どんな状況で・何を実現したいか」という形式で記述し、新しい筆記具のコンセプトを検討する際の判断軸として機能させた。
・負のサイクルが解消された状態を体験の流れとして描くことで、目指すべきUXの方向性をチーム内で共有可能な形に整えた。

## ■ A6｜新製品・新事業の企画提案能力

### 【提案の構築】
・利用状況把握・課題定義・ユーザーストーリーを一連の根拠として統合し、「新しい筆記具とは何か」という問いに対する答えをユーザーの実態に基づいたコンセプトとして構築した。
・提案内容は機能要件の列挙ではなく、「誰の・どんな体験上の問題を・どう解決するか」という構造で整理し、意思決定者が判断できる形式に仕上げた。

### 【事業提案の場でのプレゼンテーション】
・社内の事業提案の場において、上長・役員層に対してプレゼンテーションを実施した。
・筆記具ユーザーの利用実態と課題構造を可視化した資料をもとに説明することで、「なぜこの方向性が必要か」の根拠をユーザーの行動事実から示した。
・提案に対して合意を得るとともに、次フェーズへの移行判断を引き出した。

## ■ A7｜ユーザー要求仕様作成能力

### 【課題発生箇所の定義】
・筆記具を使う体験の流れを整理する中で、「どの場面で課題が起きるか」「どの文脈で課題が起きやすくなっているか」を体験ベースで定義した。
・課題を個別の事象として扱うのではなく、体験の構造の中に位置づけることで、潜在的な要求を含めた整理が可能になった。

### 【ユーザー要求事項の再定義】
・当初の要求仮説と、調査・分析を通じて得られた利用実態とのズレを照合し、ユーザー要求事項をまとめ直した。
・「突発的な気づきの瞬間に、摩擦なく使い始められること」など、利用文脈に根ざした要求として再定義することで、機能仕様に落ちる前の段階で「何のための要求か」が明示された状態を整えた。
・再定義した要求事項は、後続の設計解決案・評価フェーズにおける判断基準として共有した。
