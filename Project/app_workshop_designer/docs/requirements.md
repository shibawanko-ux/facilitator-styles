# ワークショップデザイナー 要件定義

**作成日**: 2026-03-12

---

## 概要

Claude Codeとの対話を通じてワークショップを設計し、アウトプットをファイルとして残すツール。
CLIアプリではなく、Claude Codeのチャット上で完結する。

---

## 背景・課題

- ワークショップ設計の壁打ちをClaude.aiで行っていたが、会話が消えてアウトプットが残らない
- 与件整理・タイムライン・スライド構成などを毎回作り直している
- プロジェクトごとにフォルダで管理したい

---

## 利用シーン

1. クライアントから与件を受け取った後、整理・精査する
2. ワーク内容・問いの設計を壁打ちする
3. タイムラインを作成する
4. 当日スライドの骨子を作成する

---

## 主な機能

| 機能 | 説明 |
|------|------|
| 与件整理 | クライアントの与件を構造化してファイルに保存 |
| 壁打ち | タイムライン・ワーク・問いの設計をAIと対話しながら検討 |
| ファイル保存 | 対話で決まった内容をMarkdownファイルに保存 |
| 資料自動格納 | 添付されたファイルは即座にinputsフォルダへコピー（確認不要） |
| プロジェクト管理 | クライアントごとにフォルダを作成して管理 |

---

## アウトプット

```
projects/YYYYMMDD_クライアント名/
├── inputs/          ← 添付資料
└── outputs/
    ├── 01_与件整理.md
    ├── 02_要件定義.md
    ├── 03_タイムライン.md
    └── 04_スライド構成.md
```

---

## 技術構成

- **実行環境**: Claude Code（ターミナル）
- **AIモデル**: Claude（claude-sonnet-4-6）
- **保存形式**: Markdown
