# スキル: リファクタリング

## このスキルの使い方
「このコードをリファクタリングして」と依頼するとこの手順で実行される。

---

## リファクタリング手順

### 1. 現状把握（変更前に必ず実施）
- 対象ファイルを読む
- 何をしているコードか理解する
- テストが存在するか確認する

### 2. リファクタリングの方針確認
変更前にユーザーに確認する：
- どの部分を改善したいか
- 動作を変えずに整理するだけか、設計も見直すか

### 3. 変更ルール
- **一度に大きく変えない**（小さく分けて変更する）
- **動作を変えない**（リファクタリングはバグ修正ではない）
- **変更箇所をコメントで説明しない**（コード自体をわかりやすくする）

### 4. よくあるリファクタリングパターン

#### 重複コードの削除
```python
# Before: 同じ処理が2箇所
def process_a():
    data = load_csv()
    ...

def process_b():
    data = load_csv()  # 重複！
    ...

# After: 共通関数に切り出す
def load_data():
    return load_csv()
```

#### 関数の分割
```python
# Before: 1つの関数が大きすぎる（30行以上）
def do_everything():
    # データ取得
    # データ加工
    # レポート生成
    # ファイル保存
    ...

# After: 責務ごとに分割
def fetch_data(): ...
def process_data(data): ...
def generate_report(data): ...
def save_report(report): ...
```

### 5. 完了確認
- 変更前と同じ動作をするか確認
- テストがあれば実行する
