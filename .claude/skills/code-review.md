# スキル: コードレビュー

## このスキルの使い方
「このファイルをコードレビューして」と依頼するとこの手順で実行される。

---

## レビュー手順

### 1. まずファイルを読む
- 対象ファイルを全て読む
- 関連ファイルも確認する（import先など）

### 2. チェック項目

#### セキュリティ
- [ ] SQLインジェクションのリスクはないか
- [ ] ユーザー入力をそのまま使っていないか
- [ ] APIキーや秘密情報がコードに直書きされていないか

#### コード品質
- [ ] 同じ処理が重複していないか（DRY原則）
- [ ] 関数・変数名がわかりやすいか
- [ ] 1つの関数が複数の責務を持っていないか

#### Python/Flask固有
- [ ] エラーハンドリングが適切か
- [ ] レスポンスのステータスコードが正しいか

#### TypeScript/React固有
- [ ] 型定義が適切か（`any` を多用していないか）
- [ ] useEffectの依存配列が正しいか
- [ ] 不要な再レンダリングが起きていないか

### 3. フィードバックの形式
問題点を重大度ごとに分類して報告：
- **Critical（必須修正）**: セキュリティ・バグ
- **Major（推奨修正）**: 品質・パフォーマンス
- **Minor（任意）**: スタイル・読みやすさ
