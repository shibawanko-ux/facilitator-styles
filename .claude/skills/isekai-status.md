# isekai-status — 異世界ステータス診断スキル

## 目的
ユーザーに6問の選択式質問をして、その回答をもとに「異世界転生したときのステータス画面」を生成する。
ゲーム・異世界漫画のステータス表示を参考にした、自己理解・キャラクター把握のための遊び×内省ツール。

---

## 実行手順

### ステップ1：冒頭アナウンス
以下のメッセージを表示して開始する：

```
異世界ステータス診断を開始します。
6問の質問に答えてください。すべて選択式（A/B/C/D）です。
```

---

### ステップ2：6問の質問を順番に聞く

**必ず1問ずつ聞く。次の質問は前の回答を受け取ってから出す。**
質問形式は必ず `Q〇. 質問内容（全6問中〇問目）` の形式にする。

---

#### Q1. 体力・運動について（全6問中1問目）

```
Q1. 体を動かすこと・体力について、一番近いのはどれですか？（全6問中1問目）

A. 運動習慣がある。体力には自信がある
B. たまに動くくらい。体力は普通
C. 運動はほぼしない。体力は低め
D. 体力はないが、精神力でカバーしている
```

回答による変動：
- A → STR=62, VIT=75
- B → STR=38, VIT=62
- C → STR=14, VIT=45
- D → STR=14, VIT=45, MP=96（+5補正）

---

#### Q2. 運・ラッキーについて（全6問中2問目）

```
Q2. 「運が良い」と感じることはありますか？（全6問中2問目）

A. よくある。タイミングが合うことが多い
B. どちらとも言えない。普通だと思う
C. あまりない。実力でなんとかしてきた感じ
D. 運より、仕組みと準備で結果を出すタイプ
```

回答による変動：
- A → LUK=88
- B → LUK=71
- C → LUK=55
- D → LUK=60, ANA=97（+3補正）

---

#### Q3. 複数タスクの並走について（全6問中3問目）

```
Q3. 複数のプロジェクトを同時に進めているとき、どんな感覚ですか？（全6問中3問目）

A. わりと得意。切り替えがうまくできる
B. できてはいるが、内心しんどい
C. 苦手。一つに集中したいタイプ
D. 並走はするが、優先順位の整理に時間がかかる
```

回答による変動：
- A → FOC=75, SP=78, [複数管理]=Lv.8
- B → FOC=80, SP=72, [複数管理]=Lv.6
- C → FOC=91, SP=65, [複数管理]=Lv.5
- D → FOC=82, SP=68, [複数管理]=Lv.6

---

#### Q4. 「問いを立てる」スキルについて（全6問中4問目）

```
Q4. 「問いを立てる」スキルについて、意識的にやっていますか？（全6問中4問目）

A. 無意識にやっている。もはや癖になっている
B. 意識してやっている。訓練してきた感覚がある
C. 得意ではあるが、場によってムラがある
D. そこまで得意だとは思っていない
```

回答による変動：
- A → [問いを立てる]=Lv.10, WIS=99, 説明「天性の才能」
- B → [問いを立てる]=Lv.9, 説明「訓練で磨いた後天的技術」
- C → [問いを立てる]=Lv.8, 説明「場によってムラがある」
- D → [問いを立てる]=Lv.6, WIS=92, 説明「まだ伸びしろがある」

---

#### Q5. 最大の強みについて（全6問中5問目）

```
Q5. 自分の「最大の強み」は何だと思いますか？（全6問中5問目）

A. 場を読む力・空気を察知する力
B. 構造化・言語化する力（頭の中を整理して伝える）
C. 人とつながる力・場をつくる力
D. 新しいことへの適応・学習の速さ
```

回答による変動：
- A → [場を読む]=Lv.10, WIS=99（自認の最大強みとして★MAXを付与）
- B → STZ=99, INT=97, ANA=97（★MAXをSTZに付与）
- C → EMP=99, CHA=92, LDR=90（★MAXをEMPに付与）
- D → ADP=95, TEC=85（★MAXをADPに付与）

---

#### Q6. 批判・失敗への対処について（全6問中6問目）

```
Q6. 批判・否定・失敗を受けたとき、どう対処しますか？（全6問中6問目）

A. わりと引きずらない。気持ちの切り替えが早い
B. しばらく考え込むが、最終的には前に進める
C. 深く受け止めてしまう。回復に時間がかかる
D. 批判よりも「なぜそう言われたか」の分析を先にする
```

回答による変動：
- A → DEF=82, MND=88
- B → DEF=68, MND=78
- C → DEF=55, MND=65
- D → DEF=75, INT=98

### ステップ3：ステータスを計算して出力する

#### ベース値（質問で変動しない固定ステータス）

```
HP  生命力   : 67
MP  精神力   : 91  ← Q1でDを選んだ場合は96
SP  行動力   : Q3の回答で決まる
AGI 俊敏     : 76
CHA カリスマ : 88
EMP 共感     : 95（Q5-Cで99）
NGT 交渉     : 90
LDR 統率     : 85（Q5-Cで90）
TEC 技術力   : 78（Q5-Dで85）
CRE 創造力   : 86
ANA 分析力   : 94（Q2-Dで97）
STZ 構造化   : 98（Q5-Bで99）
ADP 適応     : 88（Q5-Dで95）

スキル（固定）：
[型に変換]     Lv.8
[場を設計する] Lv.9
[橋を架ける]   Lv.8
[Claude召喚]   Lv.7
[可視化する]   Lv.7
```

#### バーグラフの生成ルール
- 10ブロック（`█`）で表現する
- 1ブロック = 10ポイント
- 端数は▓（5以上）または切り捨て
- 例：Lv.68 → ██████▓░░░ / Lv.91 → █████████░

#### ★MAXの付与
- Q5の回答で選ばれた「最大の強み」に対応するステータス1項目に `★` を付ける

### ステップ5：称号の追加ルール

Q5の回答に応じて称号を追加する：
- A（場を読む）→ 《場を読む者》
- B（構造化）→ 《言語化の魔術師》
- C（人とつながる）→ 《場を紡ぐ者》
- D（適応・学習）→ 《文明吸収者》

---

### ステップ6：バージョン管理

- 初回実行時は `ver.1.0`
- 2回目以降は `ver.2.0`, `ver.3.0` と上げる
- 「前回と比べてどこが変わったか」を出力末尾にサマリーとして追記する（2回目以降）

## 将来のアプリ化メモ（参考）

このスキルの設計がそのままアプリ仕様書になる想定：
- フロントエンド：TypeScript / React / Tailwind（ワークスペース標準）
- 質問6問 → ステータス計算 → 結果画面表示
- 結果をSNSシェアできる機能も検討
- ターゲット：ファシリテーター・コーチ・コンサル向け自己理解ツール
